Überblick

Managed System Software kann auf verschiedene Arten installiert werden. Die CD Dell OpenManage™ PowerEdgeInstallation and Server Management bietet Installations-IPS-Skripts zur Installation, Aktualisierung und Deinstallation von Server Administrator und anderen Managed System Softwarekomponenten auf Ihrem verwalteten System. Zusätzlich können Sie den Server Administrator mittels verwalterter Systemsoftware einer unbeaufsichtigten Installation über ein Netzwerk auf mehreren System installieren.

ANMERKUNG: Eine Liste aller aktuell unterstützten Systeme erhalten Sie in der Datei readme_ins.txt auf der CD Dell PowerEdge Installation and Server Management. Wenn eine vorherige Version der Systems Management Software entdeckt wird, müssen Sie sie zuerst deinstallieren. Nachdem die Deinstallation abgeschlossen ist, muss das System neu gestartet werden, bevor die neueste Version der Systems Management Software installiert werden kann.

Unbeaufsichtigte und nach Skript ablaufende Installation im Hintergrundmodus

Sie können die CD Dell PowerEdge Installation and Server Management zur Durchführung einer nicht überwachten und nach Skript im Hintergrund stattfindenden Installation des Server Administrators auf Systemen verwenden, die unterstützte Novell® Netware®-Betriebssysteme ausführen.

Bevor Sie beginnen

  • Lesen Sie die Installationsvoraussetzungen, um sicherzustellen, dass Ihr System die Mindestanforderungen erfüllt.

  • Lesen Sie das Server Administrator: Kompatibilitätshandbuch. Dieses Dokument enthält Kompatibilitätsinformationen zu Server Administrator-Installation und -Betrieb auf verschiedenen Hardwareplattformen, die unterstützte Microsoft® Windows®, NetWare-, und Red Hat® Enterprise Linux-Betriebssysteme ausführen.

  • Lesen Sie die Dell OpenManage-Installationsdatei readme_ins.txt auf der CD Dell PowerEdge Installation and Server Management. Die Datei enthält die neuesten Informationen zu neuen Funktionen, Lösungen und Hardwareanforderungen sowie Informationen zu bekannten Problemen.

  • Lesen Sie die Infodatei des Server Administrator auf der CD Dell PowerEdge Installation and Server Management. Die Datei enthält die neuesten Informationen über Software-, Firmware- und Treiberversionen sowie Informationen über bekannte Probleme.

  • Lesen Sie die Installationsanweisungen für Ihr Betriebssystem.

Installationsvoraussetzungen

In den folgenden Abschnitten werden die allgemeinen Voraussetzungen für die Installation von Managed System Software beschrieben.

Unterstützte Betriebssystemversionen

Die verwaltete Systemsoftware Server Administrator unterstützt das folgende Betriebssystem minimal:

  • Novell NetWare Version 6.5

ANMERKUNG: In der Infodatei von Server Administrator auf der CD Dell PowerEdge Installation and Server Management oder im Dell OpenManage Server Administrator-Kompatibilitätshandbuch auf der Dokumentations-CD finden Sie die aktuellste detaillierte Liste der Server Administrator-Dienste, die auf jedem unterstützten Betriebssystem unterstützt werden.

Systemanforderungen

Die verwaltete Systemsoftware muss auf jedem zu verwaltenden System installiert werden. Dann können Sie jedes System verwalten, indem Sie Server Administrator lokal oder entfernt über einen unterstützten Web-Browser ausführenManaged System Software.

Anforderungen für das Managed System

  • Eine der unterstützten Betriebssystemversionen.

  • Mindestens 64 MB RAM.

  • Mindestens 256 MB an freier Festplattenspeicherkapazität.

  • Administratorrechte.

  • Eine TCP/IP-Verbindung zum überwachten System und zum Remote-System zur Vereinfachung der Verwaltung des Remote-Systems.

  • Der einfache Netzwerkverwaltungsprotokoll (SNMP) -Standard.

  • Maus, Tastatur und Monitor zur lokalen Verwaltung eines Systems. Für den Monitor ist eine Mindestauflösung von 800 x 600 erforderlich. Die empfohlene Bildschirmauflösung ist 1 024 x 768.

  • Der RAS-Dienst des Server Administrator erfordert, dass ein Remote Access Controller (RAC) auf dem zu verwaltenden System installiert wird. Vollständige Software- und Hardwareanforderungen finden Sie im Dell Remote Access Controller 4-Benutzerhandbuch oder im Dell Embedded Remote Access Controller/MC-Benutzerhandbuch

Unterstützte Systemverwaltungsprotokollstandards

Ein unterstützter Systemverwaltungsprotokollstandard muss vor der Installation der verwalteten Systemsoftware installiert sein.Auf unterstützten NetWare-Betriebssystemen unterstützt die verwaltete Systemsoftware nur den SNMP-Systemverwaltungsstandard. SNMP ist immer installiert. Die CIM- und WMI-Standards sind nicht verfügbar.

ANMERKUNG: Informationen zur Installation eines unterstützten Systemverwaltungsprotokollstandards auf Ihrem verwalteten System finden Sie in der Dokumentation zum Betriebssystem.

Installationsverfahren

In diesem Abschnitt wird erklärt, wie Managed System Software auf einem System mit unterstütztem Netware-Betriebssystem installiert, aufgerüstet und deinstalliert wird.

Managed System Software installieren und aktualisieren

In diesem Abschnitt wird erklärt, wie die Software mithilfe der folgenden Installationsoptionen installiert und aktualisiert wird.

  • Verwenden Sie das Setup-Programm auf der CD Dell PowerEdge Installation and Server Management zur Installation oder zur Aktualisierung des Server Administrators und anderer Managed System Software.

  • Verwenden Sie das nach Skript im Hintergrundmodus ablaufende Installationsverfahren zur Installation des Server Administrators und anderer Managed System Software auf mehreren Systemen.

Voraussetzungen zur Installation oder Aktualisierung von Managed System Software

Es folgen die Voraussetzungen für die Installation der Managed System Software:

  • Administrator-Zugriffsrechte müssen vorhanden sein.

  • Der mit dem Betriebssystem gelieferte SNMP-Agent muss auf dem verwalteten System installiert sein. Wenn Sie unterstützende Agenten für den SNMP-Agenten verwenden wollen, müssen Sie die Betriebssystemunterstützung für den SNMP-Standard vor dem Server Administrator installieren. Weitere Informationen über die Installation von SNMP entnehmen Sie den Installationsanweisungen für das auf Ihrem System ausführende Betriebssystem.

ANMERKUNG: Managed System Software -Installation wird auf Systemen nicht unterstützt, die eine frühere Version der Systems Management Software als Version 4.3 enthalten. Wenn eine vorherige Version der Systems Management Software entdeckt wird, müssen Sie zuerst die Systems Management Software auf dem System deinstallieren.

Managed Systems Software mit der CD Dell PowerEdge Installation and Server Management installieren und aktualisieren.

  1. Legen Sie die Dell PowerEdge-CD Installation and Server Management in das CD-Laufwerk des Systems ein.

Die CD wird automatisch geladen, und alle Datenträgernamen werden angezeigt.

Wenn der Name des Datenträgers nicht angezeigt wird, geben Sie volume an der Konsoleneingabeaufforderung ein und drücken <Eingabe>.

  1. Notieren Sie die Datenträgerbezeichnung für das CD-Laufwerk.

  2. Geben Sie load nwconfig ein und drücken Sie <Eingabe>.

  3. Wählen Sie auf dem Bildschirm NetWare-Konfiguration unter Konfigurationsoptionen Produktoptionen und drücken Sie <Eingabe>.

  4. Wählen Sie im Menü Weitere Installationsvorgänge die Option Nicht aufgeführtes Produkt installieren und drücken Sie <Eingabe>.

ANMERKUNG: Es ist möglich, dass nach dem Drücken von <Eingabe> ein Dialogfeld geöffnet wird, das früher gewählte Pfade auflistet. Drücken Sie <Esc>, um dieses Dialogfeld zu schließen.

Eine Standardmeldung kündigt dann an, dass das Produkt von Laufwerk A: installiert werden muss.

  1. Drücken Sie <F3>, um einen anderen Installationspfad zu bestimmen.
    Löschen Sie A: und geben Sie Datenträgername:srvadmin\netware ein, wobei Datenträgername der Name des dem CD-Laufwerk zugeteilten Datenträgers ist.

ANMERKUNG: Sie können keine Leerstellen im Pfadnamen eingeben.
  1. Nach Eingabe des Pfades drücken Sie <Eingabe>, um fortzufahren.
    Eine Meldung wird angezeigt, die besagt:
    Indicate which file groups you want installed.

    (Geben Sie an, welche Dateigruppen Sie installieren wollen.)

  2. Wählen Sie eine beliebige andere Komponente aus, die Sie installieren möchten. Server Administrator ist nicht in der Liste von Dateigruppen enthalten, da die Komponente standardmäßig installiert ist.

ANMERKUNG: Zur Installation vorgemerkte Elemente sind durch ein X gekennzeichnet. Drückt man neben einem Element auf <Eingabe>, wird dieses ausgewählt. Drückt man die Leertaste neben einem Element, wird die Einstellung zwischen ausgewählt und nicht ausgewählt umgeschaltet.
  1. Drücken Sie <F10>, um die ausgewählten Komponenten zu akzeptieren.

  2. Der Zusammenfassungs-Bildschirm wird als nächstes angezeigt. Prüfen Sie die Zusammenfassungsinformationen und drücken Sie <Eingabe>, um fortzufahren, oder <Esc>, um die Installation abzubrechen.
    Die Installation des Server Administrators und der gewählten Komponenten beginnt.

  3. Der Bildschirm Systems Management-Sicherheitsdienstprogramm wird geöffnet und fragt, ob Sie die Standardsicherheitseinstellungen für SNMP-Set-Vorgänge ändern wollen.

  4. Wählen Sie eine der folgenden Optionen:

    • Drücken Sie <j>, um das Standardkennwort für root zu ändern.
      Geben Sie das neue Kennwort ein und wiederholen Sie dann die Eingabe zur Bestätigung.

ANMERKUNG: Das Standardkennwort für root ist calvin.
    • Drücken Sie <n>, um das Sicherheitsdienstprogramm zu verlassen und das Kennwort für root zu seinem Standardwert zurückzusetzen.

Das Systems Management-Sicherheitsdienstprogramm wird geöffnet.

  1. Drücken Sie bei entsprechender Aufforderung auf <y>, um SNMP-Set (Write-) Vorgänge zuzulassen, SNMP-Set-Vorgänge (Nur-Lesen) nicht zuzulassen, <n>oder eine beliebige Taste, um fortzufahren.

Das Installationsverfahren wird fortgesetzt, und Statusmeldungen werden angezeigt. Wenn die Installation abgeschlossen ist, wird ein Dialogfeld geöffnet, das den Abschluss der Installation anzeigt.

  1. Drücken Sie <Eingabe>.

  2. Drücken Sie zweimal <Esc> und einmal <Eingabe>, um zur Konsoleneingabeaufforderung zurückzukehren und das NWConfig-Programm zu beenden.

Eine Installation oder eine Aktualisierung der Manged System Software nach Skript im Hintergrundmodus ausführen

Verwenden Sie das folgende Verfahren zur Erstinstallation der Managed System Software bzw. für die Aktualisierung zu einer neueren Version der Manages System Software ohne Eingaben durch den Benutzer.

  1. Legen Sie die Dell PowerEdge-CD Installation and Server Management in das CD-Laufwerk des Systems ein.

Die CD wird automatisch geladen, und der Datenträgernamen wird angezeigt.

Wenn der Name des Datenträgers nicht angezeigt wird, geben Sie volume an der Konsoleneingabeaufforderung ein und drücken <Eingabe>, um alle Datenträgernamen anzuzeigen.

  1. Notieren Sie die Datenträgerbezeichnung für das CD-Laufwerk.

  2. Geben Sie Folgendes ein, und drücken Sie auf <Enter>:

load volume_name:\srvadmin\netware\ominss32.nlm [-noinstallweb] [-noam] [-norac]

Das Installationsskript installiert alle erforderlichen Dateien auf das verwaltete System.

Die Lizenzvereinbarung (license.txt) befindet sich auf der CD Dell PowerEdge Installation and Server Management im Verzeichnis root.

Die Einstellung –noinstallweb ist optional und wird zu einer Installation führen, die nur die Befehlszeilenkomponenten hat und nicht über einen Browser zugänglich sein wird.

Die Einstellung –noam ist optional und wird zu einer Installation führen, die keinen Dell OpenManage Array Manager hat.

Die Einstellung –norac ist optional und wird zu einer Installation führen, die keinen Dell Remote Access Controller-Softwareunterstützung hat.

Managed System Software deinstallieren

Sie können die Managed System Software über das Menü Produktoptionen im Dienstprogramm NWConfig vom System deinstallieren oder über das Hintergrund-Deinstallations-Skript auf der CD Dell PowerEdge Installation and Server Management.

Deinstallation mittels NWConfig-Dienstprogramm

  1. Geben Sie an der NetWare-Befehlszeilenkonsole load nwconfig ein und drücken Sie <Eingabe>.

  2. Wählen Sie Produktoptionen im Menü Konfigurationsoptionen.

  3. Wählen Sie Ansicht/Konfigurieren/Entfernen aus dem Menü Weitere Installationsvorgänge.

  4. Wählen Sie SRVADMIN im Menü Derzeit installierte Produkte.

  5. Drücken Sie <Entfernen> um Server Administrator zu deinstallieren.

  6. Drücken Sie zweimal <Esc> und einmal <Eingabe>, um zur Konsoleneingabeaufforderung zurückzukehren und das NWConfig-Programm zu beenden.

Deinstallation ohne Eingaben durch den Benutzer durchführen

Geben Sie bei der NetWare-Befehlszeilenkonsole omunins.ncf ein und drücken Sie <Eingabe>. Das Deinstallationsskript deinstalliert alle Managed System Software-Dateien aus dem verwalteten System.

Anmerkung zur Inbetriebnahme

Instrumentation Service bei der Inbetriebnahme

Auf verwalteten Systemen, die ein NetWare-Betriebssystem ausführen, lädt der Instrumentation Service mehrere NetWare-ladbare Module (NLMs) bei Inbetriebnahme des Systems. Einige NLMs werden möglicherweise automatisch deinstalliert werden, wenn sie auf einem bestimmten System nicht erforderlich sind. Das Verfahren zur Deinstallation der NLMs erfolgt etwa zwei Minuten nach dem Systemstart oder -neustart. Während dieses Deinstallationsverfahrens wird eine Modul wird entladen-Meldung zur NetWare-Systemkonsole gedruckt.
