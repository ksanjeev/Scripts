概览

管理系统软件的安装方法有两种，可以采用其中一种方法进行安装。Dell™ PowerEdge™ Installation and Server Management CD 提供了安装脚本和 RPM 软件包，可用于在管理系统上安装、升级和卸载 Dell OpenManage™ Server Administrator 和其它 Managed System 软件组件。此外，您还可以在无人值守的情况下，通过网络在多个系统上安装 Server Administrator。

第一种安装方法使用提供的自定义安装脚本 srvadmin-install.sh。此脚本允许执行无人值守快速安装和自定义、无人值守或交互安装。通过在 Linux 脚本中加入 srvadmin-install.sh 脚本，可以在一个或多个系统上以有人或无人值守模式本地或通过网络安装 Server Administrator。

第二种安装方法使用自定义目录中提供的 Server Administrator RPM 软件包和 Linux rpm 命令。这种方法允许自定义交互安装。可以编写 Linux 脚本，由脚本以无人值守安装模式本地或通过网络在一个或多个系统上安装 Server Administrator。

建议不要将两种安装方法组合使用，这样可能会需要使用 Linux rpm 命令手动安装自定义目录中提供的必需的 Server Administrator RPM 软件包。

注：请参阅 Dell PowerEdge Installation and Server Management CD 的 readme_ins.txt 文件了解目前所支持系统的列表。

无人值守和脚本化无提示安装

可以使用 Dell PowerEdge Installation and Server Management CD 通过命令行（使用 RPM 软件包），在运行支持的 Red Hat Enterprise Linux 和 SUSE Linux Enterprise Server 操作系统的系统上以无人值守和脚本化无提示安装方式安装 Managed System Software。

开始之前

  • 阅读安装要求，以确保您的系统可满足或超出最低要求。

  • 阅读《Dell OpenManage Server Administrator 兼容性指南》。此指南包含有关在运行支持的 Microsoft® Windows®、Red Hat Enterprise Linux 和 SUSE Linux Enterprise Server 操作系统的各种硬件平台上安装和运行 Server Administrator 的兼容性信息。
  • 请阅读 Dell PowerEdge Installation and Server Management CD 上的 Dell OpenManage 安装 readme_ins.txt 文件。 除有关已知问题的信息外，该文件还包含有关新功能、修复和硬件要求的最新信息。

  • 阅读 Dell PowerEdge Installation and Server Management CD 上的 Server Administrator 自述文件。 除有关已知问题的信息外，该文件还包含有关软件、固件及驱动程序版本的最新信息。

  • 阅读操作系统的安装说明。

安装要求

以下各节介绍了 Managed System Software 的一般要求。

支持的操作系统版本

Managed Systems Software 至少在以下各个操作系统上运行：

  • Red Hat Enterprise Linux AS、ES 和 WS（版本 3 ）x86 Update 6

  • Red Hat Enterprise Linux AS、ES 和 WS（版本 3）for Intel® Extended Memory 64 Technology (Intel EM64T) Update 6
  • Red Hat Enterprise Linux AS、ES 和 WS（版本 4）for Intel x86

  • Red Hat Enterprise Linux AS、ES 和 WS（版本 4）for Intel EM64T

  • SUSE Linux Enterprise Server（版本 9）SP3 for Intel EM64T
注：请参阅 Dell PowerEdge Installation and Server Management CD 上的 Server Administrator 自述文件或 Documentation CD 上的《Dell OpenManage Server Administrator 兼容性指南》，以了解每种支持的操作系统上支持的 Server Administrator 服务的最新详细列表。

系统要求

Managed System Software 必须安装在每个要管理的系统上。 随后可以通过支持的 Web 浏览器在本地或远程管理运行 Managed System Software 的各个系统。

管理系统要求

  • 一个支持的操作系统版本。

  • 至少 64 MB RAM。

  • 至少 256 MB 可用硬盘驱动器空间。

  • 管理员权限。

  • 监测的系统和远程系统间的 TCP/IP 连接，以便于远程系统管理。

  • 简单网络管理协议 (SNMP)。

  • 鼠标、键盘和显示器，用于在本地管理系统。 显示器要求屏幕分辨率最低为 800 x 600。 建议的屏幕分辨率设置为 1024 x 768。

  • Server Administrator Remote Access Service 要求在要管理的系统上安装 Remote Access Controller (RAC)。

注：请参阅《Dell Remote Access Controller 4 用户指南》、《Dell Remote Access Controller 5 用户指南》或《Dell Embedded Remote Access Controller/MC 用户指南》，以了解详细的软件和硬件要求。
  • Server Administrator Storage Management Service 要求将 Dell OpenManage Storage Management 安装在系统上才能正确管理。 请参阅《Dell OpenManage Server Administrator 用户指南》了解软件和硬件的完整要求。

支持的系统管理协议标准

安装 Server Administrator 之前，管理系统中必须已安装支持的系统管理协议标准。 在支持的 Red Hat Enterprise Linux 和 SUSE Linux Enterprise Server 操作系统中，Server Administrator 仅支持 SNMP 系统管理标准。您必须安装操作系统附带的 SNMP 软件包。 CIM 和 WMI 不可用。

注：有关在管理型系统中安装支持的系统管理协议标准的信息，请参阅操作系统说明文件。

安装步骤

本节介绍如何在运行支持的 Red Hat Enterprise Linux 或 SUSE Linux Enterprise Server 操作系统的系统上安装、升级和卸载 Server Administrator。Server Administrator 可以使用脚本或 RPM 命令从 Dell PowerEdge Installation and Server Management CD 安装和升级。

此外，Server Administrator 包括了动态内核支持 (DKS)，这是一种在 Server Administrator 检测到其预建的设备驱动程序都不支持该内核时自动为运行内核建立设备驱动程序的功能。 本节包括以下主题：

  • 动态内核支持 (DKS)

  • OpenIPMI 设备驱动程序
  • 安装和升级 Managed System Software

  • 执行 Managed System Software 的无人值守安装

  • 从以前的版本升级
  • 卸载 Server Administrator

软件许可协议

Red Hat Enterprise Linux 和 SUSE Linux Enterprise Server 版本的 Dell OpenManage 软件的软件许可证位于 CD 的根目录上。请阅读 license.htm 文件。 安装或复制此 CD 上的任何文件，就表示您同意此文件中的条款。 此文件也会被复制到软件树的根目录（选择在其上安装 Dell OpenManage 软件）。

动态内核支持 (DKS)

Server Administrator 为 Dell PowerEdge Installation and Server Management CD 上 Server Administrator 自述文件中列出的预编译内核提供了预编译设备驱动程序。 如果运行的内核不为自述文件中列出的预编译内核之一，或者在没有预编译 Server Administrator 设备驱动程序支持重新编译内核的情况下重新配置和重新编译了所运行的内核，则 Server Administrator 可能需要使用其 DKS 功能支持运行的内核。

如果在 Server Administrator 设备驱动程序启动期间看到以下信息，Server Administrator 就会尝试使用其 DKS 功能，但是由于不满足某些前提条件而不能使用此功能：

Building <driver> using DKS... [FAILED]

where <driver> is dcdbas or dell_rbu

（正在使用 DKS 建立<驱动程序>... [失败]

其中<驱动程序>是 dcdbas 或 dell_rbu）

注：Server Administrator 将信息记录到 /var/log/messages 日志文件中。

要使用 DKS，应标识正在运行的内核，然后检查是否满足 DKS 前提条件。

确定运行的内核

  1. 以 root 身份登录。

  2. 在控制台上键入以下命令并按 <Enter>：

uname -r

系统会显示标识运行内核的信息。 如果不是 Managed System Software 自述文件中列出的，则 Managed System Software 可能需要使用 DKS 支持它。

动态内核支持前提条件

要使 Managed System Software 使用 DKS，必须符合以下相关性才能启动 Server Administrator。

  • 运行的内核必须已启用可载入模块支持。

  • /lib/modules/`uname Cr`/build 中必须有用于为运行内核建立内核模块的源。 在运行 Red Hat Enterprise Linux（版本 3）和 SUSE Linux Enterprise Server（版本 9）的系统上，kernel-source RPM 提供了必要的内核源代码。在运行 Red Hat Enterprise Linux（版本 4）的系统上，kernel*-devel RPM 提供了用于构建内核模块所需的内核源。

  • 必须安装 GNU make 公用程序。 make RPM 提供该公用程序。

  • 必须安装 GNU C 编译器 (gcc)。 gcc RPM 提供该编译器。

  • 必须安装 GNU 链接器 (ld)。 binutils RPM 提供该链接器。

满足这些前提条件后，DKS 就会在 Server Administrator 启动期间根据需要自动建立设备驱动程序。

注：不支持的内核是预编译设备驱动程序不支持的内核。 如果正在运行支持的内核，请参阅“安装和升级 Managed System Software”。

在安装 Server Administrator 后使用动态内核支持

要启用 Server Administrator 以支持预编译设备驱动程序不支持且在 Server Administrator 安装后载入的内核，请执行以下步骤：

  1. 确保在要管理的系统上满足 DKS 前提条件。

  2. 引导系统中的新内核。

Server Administrator 在载入内核后首次启动时将为系统中运行的内核建立设备驱动程序。 默认情况下，Server Administrator 将在系统启动期间启动。

将动态建立的设备驱动程序复制到运行相同内核的系统

Server Administrator 为运行的内核动态建立设备驱动程序时，会将设备驱动程序安装到 /lib/modules/<内核>/kernel/drivers/firmware 目录，其中<内核>是内核名称（通过键入 uname -r 返回）。如果系统运行着为其建立设备驱动程序的内核，则可以将新建立的设备驱动程序复制到其它系统的 /var/omsa/dks/<内核> 目录，以供 Server Administrator 使用。此操作使 Server Administrator 能够在多个系统上使用 DKS，而不用在各个系统上安装内核源。

以下情况就是一个示例：系统 A 正在运行一个内核，但其中一个 Server Administrator 预编译设备驱动程序不支持该内核。 系统 B 正在运行相同的内核。 要在系统 A 中建立设备驱动程序并将其复制到系统 B 以供 Server Administrator 使用，请执行以下步骤：

  1. 确保在系统 A 上满足 DKS 前提条件。

  2. 在系统 A 中启动 Server Administrator。

在启动期间，Server Administrator 将为系统 A 中运行的内核建立设备驱动程序。

  1. 在系统 A 上键入 uname -r 以确定运行内核的名称。

  2. 将系统 A /lib/modules/<内核>/kernel/drivers/firmware 目录下的 dcdbas.* 或 dell_rbu.* 文件复制到系统 B /var/omsa/dks/<内核>目录，其中<内核>是在步骤 3 中通过键入 uname -r 返回的内核名称。
注：/lib/modules/<内核>/kernel/drivers/firmware 目录可能包含一个或多个以下文件：dcdbas.* 或 dell_rbu.*
注：可能需要在系统 B 上创建 /var/omsa/dks/<内核> 目录。例如，如果内核名称是 1.2.3-4smp，则可以通过键入以下命令创建目录：mkdir -p /var/omsa/dks/1.2.3-4smp
  1. 在系统 B 中启动 Server Administrator。

Server Administrator 会检测到已经复制到 /var/omsa/dks/<内核> 目录的设备驱动程序支持运行的内核，并使用该设备驱动程序。

注：升级 Server Administrator 时，如果新版本的 Server Administrator 不支持带有预编译设备驱动程序的运行内核，也可使用此过程。
注：从系统 B 上卸载 Server Administrator 后，/var/omsa/dks/<内核>/*.文件，即复制到系统 B 的文件，将不会删除。如果不再需要，则必须删除这些文件。

内核受到污染时在 Red Hat Enterprise Linux Update 版本上强制启用动态内核支持

Server Administrator 为支持的 Red Hat Enterprise Linux 操作系统的“Gold”版本提供预编译的设备驱动程序。Red Hat Enterprise Linux 支持在 Update 版本上载入为“Gold”版本建立的设备驱动程序。这意味着 Server Administrator 不必为每个 Red Hat Enterprise Linux Update 版本提供预编译的设备驱动程序。在每个运行 Red Hat Enterprise Linux Update 版本的系统上，用户也不需要必须使用 DKS，才能运行 Server Administrator。然而，在 Update 版本上载入为“Gold”版本 Red Hat Enterprise Linux（版本 3）建立的设备驱动程序可能会污染内核。如果运行 Red Hat Enterprise Linux（版本 3）Update 版本的系统上的内核受到此设备驱动程序载入过程的污染，可以使用 Server Administrator 的初始化脚本命令 restart-forcekernelmatch 强制启用 DKS。DKS 将建立不会污染运行的内核的设备驱动程序。

确定运行的内核是否受到污染

启动 Server Administrator 服务后，请在 Red Hat Enterprise Linux Update 版本上执行以下步骤确定内核是否已经受到污染：

  1. 以 root 身份登录。

  2. 执行以下命令：

lsmod

如果如以下信息所示在输出信息的第一行中看到“Tainted: GF”字样，则说明运行的内核已经受到污染：

Module Size Used by Tainted: GF

（模块 使用的大小 受污染: GF）

“tainted”状态可能是由 Server Administrator 设备驱动程序载入过程造成的。

在 Red Hat Enterprise Linux Update 版本上强制启用动态内核支持

安装 Server Administrator 后，请执行以下步骤在 Red Hat Enterprise Linux Update 版本上强制启用 DKS，为运行的内核建立设备驱动程序（如果有必要），以便设备驱动程序不污染内核：

  1. 确保满足启用 DKS 的前提条件。
  2. 执行以下命令：

/etc/init.d/instsvcdrv restart-forcekernelmatch

此命令将首先停止 Server Administrator 设备驱动程序。然后，通过检查是否存在专门为与运行内核的完全同名的内核建立的预编译设备驱动程序，搜索供载入的预编译的设备驱动程序。如果找不到，它将使用 DKS 为运行的内核建立设备驱动程序。最后，该命令将重新启动 Server Administrator 设备驱动程序。

注：必须重新引导系统，清除内核的“tainted”状态。

OpenIPMI 设备驱动程序

Server Administrator 的 Server Administrator Instrumentation Service 需要具有 OpenIPMI 设备驱动程序，才能提供基于 IPMI 的信息和功能。它同时要求 OpenIPMI 设备驱动程序不能低于某个版本。要求的最低版本是根据 Red Hat Enterprise Linux 或 SUSE Linux Enterprise Server 操作系统确定的。

Server Administrator 会在三种情况下检查是否符合这一要求并强制要求满足这一要求。两次是在安装期间，一次是在运行时。这些情况为：

  • 使用 srvadmin-install.sh 外壳脚本自定义安装 Server Administrator 期间。
  • 使用 RPM 安装 Server Administrator 期间。
  • 启动 Server Administrator Instrumentation Service 时运行时间。

使用 srvadmin-install.sh 外壳脚本自定义安装 Server Administrator 期间出现错误

使用 srvadmin-install.sh 外壳脚本自定义安装 Server Administrator 期间，脚本会自动调用另一个名为 srvadmin-openipmi.sh 的外壳脚本，以确保系统上目前安装版本足够高的 OpenIPMI 设备驱动程序（例如，与 Server Administrator Instrumentation Service 要求的 OpenIPMI 设备驱动程序最低版本相同的版本或更高的版本）。如果找不到足够高的版本，srvadmin-install.sh 外壳脚本会再次调用 srvadmin-openipmi.sh 外壳脚本，尝试安装足够高的版本。如果由于某种原因安装尝试失败，则会出现适当的错误信息，其中有如何消除该错误的说明。如果有错误，必须先消除错误，然后再重新运行 srvadmin-install.sh 外壳脚本安装 Server Administrator。

使用 RPM 安装 Server Administrator 期间出现错误

为使 RPM 安装按正确顺序进行，srvadmin-hapi RPM 需要有一个 srvadmin-ipmi RPM 协助。每个 srvadmin-ipmi RPM 均用于特定的分发任务，依赖一个用于特定分发任务的 OpenIPMI RPM。除非符合 RPM 相关性，否则 srvadmin-hapi RPM 将不会安装。

在使用 RPM 安装 Server Administrator 期间， srvadmin-hapi RPM 将检查系统上目前是否安装版本足够高的 OpenIPMI 设备驱动程序。如果找不到足够高的版本， srvadmin-hapi RPM 将无法安装，并显示错误信息，提示必须安装/升级 OpenIPMI 设备驱动程序。在这种情况下，依赖 srvadmin-hapi RPM 的 RPM 也将无法安装。这时，必须再执行以下一些步骤安装/升级 OpenIPMI 驱动程序：

  1. 浏览至 srvadmin/linux/supportscripts 目录。
  2. 运行以下 srvadmin-openipmi.sh 外壳脚本：

sh srvadmin-openipmi.sh install

重新尝试安装 srvadmin-hapi RPM 和任何其它由于依赖 srvadmin-hapi RPM 而无法安装的 RPM。请执行以下步骤：

  1. 浏览至以前所在的 srvadmin/linux/custom/* 目录。
  2. 运行以下命令查看以前成功安装的 Server Administrator 软件包。

rpm -qa | grep srvadmin

该查询操作会显示所有已经安装的 Server Administrator 软件包列表。

  1. 运行以下命令，安装目录中以前没有安装的所有 RPM，因为这些 RPM 依赖以前无法安装的 srvadmin- hapi RPM：

rpm Cihv srvadmin-hapi*.rpm srvadmin-isvc*.rpm srvadmin-omhip*.rpm srvadmin-<any-other-packages>*.rpm

另外，在使用 RPM 安装 srvadmin-hapi RPM 期间，如果系统上目前已经安装针对某个版本的 OpenIPMI 驱动程序的 OpenIPMI RPM 且该驱动程序符合 Server Administrator 要求的 OpenIPMI 的驱动程序最低版本，但是，因为要求 kernel-source RPM 而目前没有安装，仍没有为给定版本的 OpenIPMI 驱动程序建立并安装 OpenIPMI 驱动程序模块，srvadmin-hapi RPM 将无法安装，并将显示错误信息。该错误信息将提示需要安装 kernel-source RPM 和建立 OpenIPMI 驱动程序。在这种情况下，依赖 srvadmin-hapi RPM 的任何 RPM 也将无法安装。此时，必须再执行以下一些步骤，安装 kernel-source RPM，并建立和安装 OpenIPMI 驱动程序模块：

  1. 对于运行 Red Hat Enterprise Linux（版本 3）的系统，从操作系统安装 CD（或者从其它一些来源）安装一份用于运行内核的适当的 kernel-source RPM。

在运行 Red Hat Enterprise Linux（版本 4）和 SUSE Linux Enterprise Server 的系统上，kernel*-devel RPM 提供了用于建立内核模块所需的源代码。

  1. 通过运行以下命令从找到或复制 RPM 的位置安装 kernel-source RPM 或 kernel*-devel RPM：

rpm Cihv kernel*.rpm

  1. 浏览至 srvadmin/linux/supportscripts 目录。
  2. 如下所示，运行 srvadmin-openipmi.sh 外壳脚本。这将建立并安装 OpenIPMI 驱动程序模块。

sh srvadmin-openipmi.sh install

重新尝试安装 srvadmin-hapi RPM 和由于依赖 srvadmin-hapi RPM 而无法安装的 RPM。

启动 Server Administrator Instrumentation Service 之后功能下降

安装 Server Administrator 之后，无论何时启动 Server Administrator Instrumentation Service，它均会执行一次 OpenIPMI 设备驱动程序运行时检查。无论何时运行 srvadmin-services.sh start 或 srvadmin-services.sh restart 命令，或者重新启动系统（在此期间会启动 Server Administrator Instrumentation Service），均会启动 Server Administrator Instrumentation Service。

尽管在目前没有安装版本足够高的 OpenIPMI 设备驱动程序的情况下，Server Administrator 安装程序将阻止 Server Administrator 的安装，但是仍然存在一种可能（虽然不常见），即在安装 Server Administrator 之后，用户能够卸载或替换版本足够高的 OpenIPMI 设备驱动程序。Server Administrator 不能防止这种情况的发生。

考虑到在安装 Server Administrator 之后用户可能会卸载或替换版本足够高的 OpenIPMI 设备驱动程序，无论何时 Server Administrator Instrumentation Service 在启动时，均会检查 OpenIPMI 设备驱动程序的版本。如果找不到版本足够高的 OpenIPMI 设备驱动程序，Server Administrator Instrumentation Service 将会降低自己的功能，不提供任何其基于 IPMI 的信息或功能。主要来讲，这意味着它将不提供任何探测器数据（例如风扇、温度或电压探测器数据）。

如果知道或怀疑系统上的 OpenIPMI 设备驱动程序版本不够高，可以执行以下步骤，运行 Dell PowerEdge Installation and Server Management CD 上的 srvadmin-openipmi.sh 外壳脚本安装版本足够高的设备驱动程序：

  1. 浏览至 srvadmin/linux/supportscripts 目录。
  2. 如下所示，运行 srvadmin-openipmi.sh 外壳脚本。

sh srvadmin-openipmi.sh install

如果不清楚系统上的 OpenIPMI 设备驱动程序的版本是否足够高，请运行 srvadmin-openipmi.sh status 命令。与 srvadmin-openipmi.sh install 命令不同，srvadmin-openipmi.sh install 是用于在必要的情况下尝试安装或升级 OpenIPMI 设备驱动程序的命令，而 srvadmin-openipmi.sh status 命令则用于显示信息，告诉您系统上的 OpenIPMI 设备驱动程序的版本是否足够高。在运行 srvadmin-openipmi.sh status 命令后，就可以决定是否需要或希望运行 srvadmin-openipmi.sh install 命令。

rpm -Uhv srvadmin-omilcore-4.3.0-1.386.rpm srvadmin-hapi-4.3.0-1.386.rpm srvadmin-deng-4.3.0-1.386.rpm srvadmin-isvc-4.3.0-1.386.rpm

  1. 运行以下命令，升级目录中因为 srvadmin-hapi RPM 以前无法升级而没有升级的所有 RPM：

rpm -Uhv srvadmin-hapi*.rpm srvadmin-isvc*.rpm srvadmin-omhip*.rpm srvadmin-<任何其它软件包>*.rpm

如果系统上目前已经安装针对某个版本的 OpenIPMI 驱动程序的 OpenIPMI RPM 且该驱动程序符合 Server Administrator 要求的 OpenIPMI 驱动程序最低版本，但是，因为要求 kernel-source RPM 而目前没有安装，仍没有为给定版本的 OpenIPMI 驱动程序建立并安装 OpenIPMI 驱动程序模块，srvadmin-hapi RPM 将无法升级，并将显示错误信息。该错误信息将提示需要安装 kernel-source RPM 和建立 OpenIPMI 驱动程序。在这种情况下，依赖 srvadmin-hapi RPM 的 RPM 也将无法升级。必须再执行以下一些步骤，安装 kernel-source RPM，并建立和安装 OpenIPMI 驱动程序模块：

  1. 对于运行 Red Hat Enterprise Linux（版本 3）的系统，从操作系统安装 CD（或者从其它一些来源）安装一份适合目前运行的内核的 kernel-source RPM。在运行 Red Hat Enterprise Linux（版本 4）和 SUSE Linux Enterprise Server 的系统上，kernel*-devel RPM 提供了用于建立内核模块所需的源代码。
  2. 通过运行以下命令从找到或复制 RPM 的位置安装 kernel-source RPM 或 kernel*-devel RPM：

rpm -ihv kernel*.rpm

  1. 浏览至 srvadmin/linux/supportscripts 目录。
  2. 如下所示，运行 srvadmin-openipmi.sh 外壳脚本。这将建立并安装 OpenIPMI 驱动程序模块。

sh srvadmin-openipmi.sh install

重新尝试升级 srvadmin-hapi RPM 和任何其它由于依赖 srvadmin-hapi RPM 而无法升级的 RPM。请执行本节所说明的步骤。

使用 srvadmin-install 外壳脚本
  1. 作为 root 登录到要求升级的运行 Red Hat Enterprise Linux 的系统。

  2. 将 Dell PowerEdge Installation and Server Management CD 插入系统上的 CD 驱动器中。 如果 CD 不自动加载，请键入 mount /dev/cdrom /mnt/cdrom。

  3. 加载 CD 后，通过键入以下某个命令浏览到 CD 上包含外壳脚本的子目录：

cd /mnt/cdrom/srvadmin/linux/supportscripts
cd /media/cdrom/srvadmin/linux/supportscripts.

  1. 运行以下脚本：

sh srvadmin-install.sh

脚本会检测 Server Administrator 的所有以前的版本。 如果装有以前的版本，会显示一条信息，指出当前的版本和所装的组件。

例如：

目前已经安装 Server Administrator 4.4.0 版。安装的组件包括；
- srvadmin-omilcore
- srvadmin-hapi
- srvadmin-deng
- srvadmin-isvc

接着，脚本会显示以下信息提示您：

Do you want to upgrade Server Administrator to 5.0?
Press ('y' for yes | 'Enter' to exit):

（是否要将 Server Administrator 升级到 5.0？按（'y' 表示是 | 'Enter' 退出）：）

  1. 选择 y 升级系统。

Server Administrator 会升级到版本 5.0。

注：在升级 Server Administrator RPM 之前，srvadmin-install.sh 外壳脚本会自动调用 srvadmin-openipmi.sh 外壳脚本，以确保系统目前安装版本足够高的 OpenIPMI 驱动程序。如果找不到足够高的版本，srvadmin-install.sh shell 脚本会调用 srvadmin-openipmi.sh 外壳脚本尝试安装足够高的版本。 如果由于某种原因安装尝试失败，则会出现适当的错误信息，其中有如何消除该错误的说明。 消除错误之后，应该重新运行 srvadmin-install.sh 外壳脚本升级 管理系统组件。

卸载 Managed System Software

可以从 Red Hat Enterprise Linux 或 SUSE Linux Enterprise Server 命令行卸载 Managed System Software。此外，您还可以在多个系统上同时执行无人值守卸载。

卸载 Managed System Software 的前提条件

必须作为 root 登录。

从 Red Hat Enterprise Linux 或 SUSE Linux Enterprise Server 命令行卸载 Managed System Software

安装 Server Administrator 时已经安装卸载脚本。可以通过键入 srvadmin-uninstall.sh 然后按 <Enter> 执行脚本。

注：执行时，srvadmin-uninstall.sh 将不会尝试卸载 OpenIPMI 驱动程序，即使在上次执行 srvadmin-install.sh 脚本或 srvadmin-openipmi.sh 脚本期间已经安装或升级 OpenIPMI 驱动程序。

自定义卸载特定组件

可卸载 Dell OpenManage 一些个别组件而无需卸载全部 Dell OpenManage。 示例如下：

要只卸载 Web Server，使用此命令：

rpm Ce `rpm -qa | grep srvadmin-iws`

要卸载诊断程序，使用此命令：

rpm Ce `rpm -qa | grep srvadmin-old`

要卸载存储，使用此命令：

rpm Ce `rpm -qa | grep srvadmin-storage`

将 Dell OpenManage 与 VMware ESX Server™ 软件一起使用

注：Dell OpenManage 与 VMware ESX Server 软件安装在一起需要特别的步骤。 这些步骤会根据 Dell OpenManage 和 ESX Server 的版本而有所不同；只支持有限的一些组合。

请参阅位于资源中心 www.dell.com/vmware 的《VMware 系统兼容性指南》来确定与此版 Dell OpenManage 兼容的 ESX Server 软件版本。每个 Dell ESX Server 版本都有相关的 Dell VMware ESX Server 部署指南，也在该 Web 位置发布。有关安装与 ESX Server 版本对应的所支持 Dell OpenManage 版本的说明均可以在该 ESX Server 版本的部署指南中找到。有关安装以后发布的所支持 Dell OpenManage 版本的说明会单独以一个明确标明的文档公布在同一位置。

使用第三方部署软件进行 Managed System Software 安装

可以使用第三方部署软件，比如 Altiris Deployment Solution，来将 Managed System Software 安装到所支持的 Dell 服务器上。 要使用 Altiris 分发和安装 Managed System Software，请启动 Altiris 应用程序并导入 Dell PowerEdge Installation and Server Management CD 的 \srvadmin\support\Altiris 目录下的 OpenManage_Jobs.bin。指定要用于导入的目标工作文件夹。可能需要修改“Run Script”（运行脚本）和“Copy File”（复制文件）任务来与部署环境相匹配。完成后，可以安排工作在 Altiris Deployment Solution 管理的支持的 Dell 系统上运行。
