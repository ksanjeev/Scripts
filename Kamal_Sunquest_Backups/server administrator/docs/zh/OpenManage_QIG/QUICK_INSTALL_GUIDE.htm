最优化 Dell™ PowerEdge™ 系统的价值

使用提供的 CD 快速设置并运行系统。 系统将会始终如一地可靠运行，无需您费心劳神。 CD 提供：

  • 简单快捷的操作系统安装 ― 通过指导您进行易于遵循的逐步安装过程，减少了安装 Microsoft® Windows®、Novell® NetWare® 和 Red Hat® Linux Enterprise 操作系统所需的时间。

Systems Management Software 概览

Dell OpenManage Systems Management Software 是一组用于 PowerEdge 系统的应用程序。此软件可以通过主动式监控、诊断、通知和远程访问等方式管理系统。

使用 Dell OpenManage 软件产品管理的每个系统均被称为管理型系统。管理型系统应用程序包括 Server Administrator、Dell OpenManage Array Manager 和 Remote Access Controller (RAC) 软件。管理站可以用来从一个中央位置远程管理一个或多个管理型系统。

安装操作系统

请执行以下步骤以确定系统中是否已安装操作系统：

  1. 确保键盘、鼠标和显示器已连接至系统，然后打开系统电源。

  2. 阅读并接受软件许可协议以继续。

系统将重新引导。 如果系统显示信息指出可引导的驱动器不存在或未找到操作系统，则表明您的系统中尚未安装操作系统。 请准备好操作系统 CD 并继续执行下面的步骤。

如果您的系统中已预安装操作系统，则无需继续此过程。 请找到系统提供的操作系统的安装说明文件，并按照此说明完成安装过程。

执行下面的步骤在系统上安装操作系统：

  1. 插入 Installation and Server Management CD 后重新启动系统。

  2. 在 Server Assistant 主页上，选择“Server Setup”（服务器设置）。

  3. 按照各步骤提供的详细说明配置硬件并安装操作系统。

有关安装 RAID 的其它信息，请参阅说明文件 CD 上的《RAID 入门指南》。

注：当您使用 Server Assistant 安装 Windows 或 Red Hat Enterprise Linux 操作系统时，Server Assistant 将相关的 Server Administrator 安装文件自动复制到硬盘驱动器上，并且在桌面上放置“Install Server Administrator”（安装 Server Administrator）和“Delete Server Administrator Installation Files”（删除 Server Administrator 安装文件）图标。
注：您可以使用“Install Server Administrator”（安装 Server Administrator）图标来安装 Server Administrator，而不使用安装 CD。在运行 Windows 的系统上，单击此图标显示标准安装界面。 在运行 Red Hat Enterprise Linux 的系统上，单击此图标运行 Red Hat Enterprise Linux 自定义安装。 如果您不想安装 Server Administrator，您可以通过单击“Delete Server Administrator Installation Files”（删除 Server Administrator 安装文件）图标来删除安装文件。您确定要继续后，所有的 Server Administrator 文件（包括图标）都会被删除。
注：当安装 Windows 操作系统时，一些装有超过 4 GB 物理内存的硬件配置在安装操作系统后需要执行附加的步骤，以充分利用所有已安装的内存。 有关物理地址扩展 (PAE) 的详情，请参阅：

www.microsoft.com/windows2000/en/advanced/help/PAE_checklist.htm

www.microsoft.com/resources/documentation/windowsserv/2003/enterprise/proddocs/en-us/paex86_2.asp

www.support.microsoft.com/default.aspx?scid=kb;en-us;283037

在支持的 Microsoft Windows 操作系统上安装 Managed System Software

  1. 使用管理员权限登录至运行支持的 Windows 操作系统的系统，以在该系统中安装 Managed System 组件。

  2. 验证系统已装有支持的浏览器。

  3. 如果系统已经安装 RAID 控制器并且您打算安装存储管理功能，请确保用于每个 RAID 控制器的设备驱动程序也已经安装。

您可以在 Service and Diagnostic Utilities CD 上找到设备驱动程序。您可以在 Windows 下验证设备驱动程序，方法是：在“My Computer”（我的电脑）上单击鼠标右键，选择“Manage”（管理），然后单击“Device Manager”（设备管理器）。

  1. 将 Installation and Server Management CD 插入到 CD 驱动器中。

安装程序应当自动启动。 如果没有自动开始，请单击“Start”（开始）按钮，再单击“Run”（运行），然后键入 x:\srvadmin\windows\setup.exe（其中 x 是 CD 驱动器的盘符）。安装程序会启动前提条件检查公用程序。

  1. 查看所执行的前提条件检查结果，然后解决好列出的所有情况。

  2. 单击“Install, Modify, Repair, or Remove Server Administrator”（安装、修改、修复或删除 Server Administrator）按钮。

  3. 在“Welcome”（欢迎）屏幕中单击“Next”（下一步）。

  4. 接受许可协议，然后单击“Next”（下一步）。

  5. 选择“Express”（快速）安装，然后单击“Next”（下一步）。

  6. 单击“Install”（安装）继续安装过程。

安装程序将根据您的硬件配置自动安装所有 Managed System Software。

  1. 安装完成后，单击“Finish”（完成），并在使用此软件之前重新启动系统（若提示）。

在支持的 Red Hat Enterprise Linux 操作系统上安装 Managed System Software

  1. 以 root 身份登录至运行支持的 Red Hat Enterprise Linux 操作系统的系统，以在该系统中安装 Managed System 组件。

  2. 将 Installation and Server Management CD 插入到 CD 驱动器中。

  3. 如果有必要，使用 mount /mnt/cdrom 命令或相似的命令装入 CD。

  4. 如果您计划安装 DRAC III、DRAC III/XT、ERA 或 ERA/O 软件，请浏览到 /srvadmin/linux/express-install-with-RAC3 目录。如果您计划安装 DRAC 4 软件，请浏览到 /srvadmin/linux/express-install-with-RAC4 目录。然后，从任一目录执行以下 RPM 命令：
    rpm Cihv *.rpm

  5. 如果不确定系统上安装了哪种 RAC，则从 /srvadmin/linux/supportscripts 目录执行 srvadmin-install.sh 脚本：
    sh srvadmin-install.sh --express
    或
    sh srvadmin-install.sh -x

此脚本将安装 RAC 软件。 如果 RAC 设备不存在，此脚本将不会安装任何 RAC 软件。

  1. 使用 sh srvadmin-services.sh start 命令启动服务。

注：您可以通过将 2>&1 | tee Ca /var/log/srvadmin.log 添加到以上的 RPM 命令中来记录 RPM 安装的输出。 所得到的命令将为：
rpm Cihv *.rpm 2>&1|tee Ca /var/log/srvadmin.log

在支持的 Novell NetWare 操作系统上安装 Managed System Software

  1. 将 Installation and Server Management CD 插入到运行 Netware 6.5 的系统的 CD 驱动器中。

CD 会自动载入。

  1. 如果卷名没有显示，请在控制台提示符下键入 volumes 并按 <Enter> 键。

将显示所有可用的卷。

  1. 记录 CD 驱动器的卷名。

  2. 输入 load nwconfig 并按 <Enter> 键。

  3. 在“NetWare Configuration”（NetWare 配置）屏幕的“Configuration Options”（配置选项）下，选择“Product Options”（产品选项）并按 <Enter> 键。

  4. 在“Other Installation Actions”（其它安装操作）菜单中，选择“Install a product not listed”（安装未列出产品）并按 <enter> 键。

注：按 <Enter> 键后，系统可能会显示一个对话框，列出先前选择的路径。 按 <Esc> 关闭此对话框。

系统将显示默认信息，说明将从驱动器 A 安装此产品：

  1. 按 <F3> 指定其它的安装路径。

  2. 删除 A: 并键入卷名和路径名：

卷名:srvadmin\netware

其中卷名是分配给 CD 驱动器的卷的名称。

注：您不能在路径名中输入空格。
  1. 键入路径名后，请按 <Enter> 继续。

以下信息会出现：

Indicate which file groups you want installed.

(请指定要安装的文件组。)

Server Administrator 不会作为选项列出，因为它是在默认情况下被安装的。

  1. 选择要安装的组件。

注：选定安装的项目旁边均带有 X。 按项目旁边的 <Enter> 键选择该项目。 按项目旁边的空格键将在选择和取消选择之间切换选择。
  1. 按 <F10> 接受选择的组件。

  2. 出现摘要屏幕，它是要停止安装的最后一点。 查看摘要信息，然后按 <Enter> 继续或按 <Esc> 终止安装。

开始安装 Server Administrator 和您所选的组件。 状况信息会出现。

Systems Management 安全性公用程序会出现，并询问您是否要更改 SNMP Set 操作的默认安全设置。

注： 默认 root 密码为 calvin。
  1. 选择以下一个选项：

    • 按 <y> 更改 root 的默认密码。

键入新密码，然后再次键入密码进行确认。

    • 按 <n> 或 <q> 退出安全公用程序，并将 root 密码重设为默认值。

继续安装 Server Administrator 和您所选的组件。 状况信息会出现。 安装完成时，一个对话框会出现，说明安装完成。

  1. 按 <Enter>。

  2. 按两下 <Esc> 再按一下 <Enter> 返回到控制台提示符，并退出 NetWare 配置程序。

注：对于 Windows，Management Station 应用程序只在运行以下操作系统的系统上运行：Windows XP Professional、Windows 2000 Professional (SP4)、Windows Server 2003 系列（包括 Small Business Server [SBS]）和 Windows 2000 Server 系列（包括 SBS），并且可以管理运行所支持的 Windows、NetWare 和 Red Hat Enterprise Linux 操作系统的系统。 IT Assistant 在 Windows Server 2003 SBS 或 Windows 2000 Server SBS 操作系统上不受支持。
注：只有 RAC 控制台和 BMC 管理公用程序在 Red Hat Enterprise Linux 操作系统上受支持。

在 Microsoft Windows 操作系统上安装 Management Station Software

注：在带有早于 4.3 版的 Systems Management Software 的系统上，不支持 IT Assistant 安装。 如果系统上已安装 4.2 或更早版本 (>=3.0) 的 Systems Management Software，则不能直接升级至 4.5 版。 要保留这些设置，必须升级至 4.3 版，然后可升级到 4.5 版以获得最新的软件。 4.3 版可在 Dell 支持站点 support.dell.com 上得到。
注：要成功安装 IT Assistant，则应在安装 Management Station Software 之前使用 Systems Management Consoles CD 在管理站上安装 MSDE 代理（如果还没安装的话）。在安装完 MSDE 后，重新引导系统。

要将应用程序安装到管理站，请执行以下步骤：

  1. 使用管理员权限登录至您想在其中安装 Management Station Applications 的系统。

  2. 将 Systems Management Consoles CD 插入 CD 驱动器中。

安装程序应当自动启动。如果没有自动启动，单击“Start”（开始）按钮，单击“Run”（运行），然后键入 x:\windows\setup.exe（其中 x 是 CD 驱动器的盘符）。安装程序会启动前提条件检查公用程序。

  1. 查看所执行的前提条件检查结果，然后解决好列出的所有情况。

  2. 单击“Install, Modify, Repair, or Remove Management Station”（安装、修改、修复或删除 Management Station）按钮。

“Welcome”（欢迎）屏幕会显示。

  1. 单击“Next”（下一步）。

“License Agreement”（许可协议）屏幕会显示。

  1. 选择“I Accept”（我接受），然后单击“Next”（下一步）。

“Setup Type”（安装类型）屏幕会显示。

  1. 选择“Express”（快速），然后单击“Next”（下一步）。

“Ready To Install”（准备安装）屏幕会显示。

  1. 单击“Install”（安装）接受所选的功能，然后开始安装。

显示“Installing Dell OpenManage Management Station”（安装 Dell OpenManage Management Station）的屏幕会出现。信息出现，说明正在安装软件组件的状况和进程。

当所选组件被安装后，“Install Wizard Completed”（安装向导已完成）对话框会出现。

  1. 单击“Finish”（完成）退出 Dell OpenManage Management Station 安装。

在 Red Hat Enterprise Linux 操作系统上安装 Management Station Software

管理站软件套件的 BMC 管理公用程序和 RAC 组件可在运行 Red Hat Enterprise Linux 的管理站上使用。

要在管理站上安装 BMC 管理公用程序，请执行下列步骤来启动安装程序：

  1. 以 root 身份登录至您想在其中安装 Management Station 组件的系统。

  2. 如果有必要，使用 mount /mnt/cdrom 命令或相似的命令装入 CD。

  3. 浏览至 /linux/bmc 目录，并执行 rpm -ivh *.rpm 命令。

要安装 RAC Management Station 组件，请执行下列步骤以启动安装程序。

  1. 以 root 身份登录至您想在其中安装 Management Station 组件的系统。

  2. 如果有必要，使用 mount /mnt/cdrom 或相似的命令装入 CD。

  3. 浏览至 /linux/rac 目录，并执行 rpm -ivh *.rpm 命令。

更新驱动程序和公用程序

Service and Diagnostic Utilities CD 包含有可以用来更新系统的设备驱动程序和公用程序。您必须将这些驱动程序和公用程序复制到软盘或硬盘才能使用它们。

要获得最新的产品更新，如 BIOS、固件、驱动程序和 Dell OpenManage 应用程序，请访问 Dell 支持网站 support.dell.com。

注：您只能在运行 Windows 的系统上运行 Service and Diagnostic Utilities CD。 要为运行 Red Hat Enterprise Linux 或 NetWare 的系统抽取驱动程序，您必须在运行 Windows 的系统上使用 Service and Diagnostic Utilities CD，然后将选定的文件复制到最终目标系统，或与最终目标系统共享选定的文件。
  1. 将 Service and Diagnostic Utilities CD 插入到运行 Windows 系统上的 CD 驱动器中。

安装程序应当自动启动。 如果没有自动开始，请单击“Start”（开始）按钮，再单击“Run”（运行），然后键入 x:\setup.exe（其中 x 是 CD 驱动器的盘符）。

  1. 选择系统，并从“Select Drivers/Utilities Set”（选择驱动程序/公用程序集）中选择您想更新的文件集，然后单击“Continue”（继续）。

  2. 从“Drivers and Utilities Page”（驱动程序和公用程序页），通过单击设备名链接，选择您想更新的公用程序或驱动程序文件。

系统提示您选择保存文件的位置。 这些文件是以压缩格式发送的。 抽取 BIOS 和固件文件时通常需要软盘。

  1. 请解压缩这个文件。

关于 BIOS、固件、驱动程序和 Dell OpenManage 应用程序版本的其它信息，请参阅《Dell OpenManage Server Administrator 兼容性指南》。

Dell PowerEdge Updates CD 仅由 Subscription Kit 提供，并且通过 Dell OpenManage Server Update Utility (SUU) 和 Dell Update Packages (DUP) 为系统提供新版软件。

  1. 将 Updates CD 插入到运行 Windows 或 Red Hat 操作系统的系统的 CD 驱动器中。

  2. Server Update Utility 应该自动启动。如果没有自动启动，请单击“Start”（开始）按钮，单击“Run”（运行），键入 cmd，然后按 Enter。当命令提示符出现时，将目录更改为 CD 驱动器的根目录。键入 suu -g（用于图形模式）或 suu -h（为显示帮助选项）。

注：在 Red Hat Enterprise Linux 系统上，只能从 X Window 系统使用 -g（图形模式）选项。
  1. 要在 GUI 中查看存储库比较，请单击左侧树中显示的服务器主机名。SUU 会将服务器上所有的组件与其存储库中可用的组件版本进行比较。对于服务器上与在 SUU 存储库中版本保持一致的各个组件，SUU 均报告一个绿色的对号图标。对于服务器中任何过时的或能使用 SUU 更新的组件，会显示一个黄色的警告图标。如果找到可更新组件，“Update”（更新）按钮就会在屏幕右上角被启用。

  2. 单击“Update”（更新）应用服务器列出的所有更新。您无法选择性地应用更新。SUU 会出现一个对话框，要求确认将更新的所列组件。单击“Yes”（是）继续。

  3. “Update Success”（更新成功）窗口会出现，确认成功更新了 SUU 和 DUP，并且可能提示您重新引导系统才能完成更新。

Dell Update Packages：

在 SUU 中单击“Update”（更新）时，SUU 将确定适用于系统的 Dell Update Packages，并会自动安装它们。

请参阅《Dell OpenManage Server Update Utility 用户指南》了解其它信息。
