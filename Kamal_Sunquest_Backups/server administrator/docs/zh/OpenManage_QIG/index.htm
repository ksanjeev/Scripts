使用 Server Assistant 安装操作系统

请执行以下步骤以确定系统中是否已安装操作系统：

  1. 确保键盘、鼠标和显示器已连接至系统，然后打开系统电源。

  2. 阅读并接受软件许可证协议以继续。

如果系统显示信息指出可引导的驱动器不存在或未找到操作系统，则表明您的系统中尚未安装操作系统。 请准备好操作系统 CD 并继续执行下面的步骤。

如果您的系统中已预安装操作系统，则无需继续此过程。 请找到系统提供的操作系统《安装说明》文件，并按照此说明完成安装过程。

执行下面的步骤，在系统上安装操作系统：

  1. 插入 Dell PowerEdge Installation and Server Management CD，然后重新启动系统。

系统引导时提供以下选项：

    • Server Assistant ― 通过 CD 引导。

    • “Server Assistant with CD media check”（Server Assistant 附带 CD 介质检查）― 检查 CD 是否正确创建，以及其内容是否有效。

    • “Skip CD boot - Boot to hard drive”（跳过 CD 引导 - 引导至硬盘驱动器）― 从硬盘驱动器引导，并验证系统是否安装了操作系统。 如果系统未安装操作系统，则通过 CD 引导。

注：如果您在 10 秒钟内未作出选择，系统会自动通过 CD 引导。
  1. 在 Server Assistant 主页面上选择 “Server Setup”（服务器设置）

  2. 按照各步骤提供的详细说明配置硬件并安装操作系统。

有关安装 RAID 的其它信息，请参阅 Documentation CD 上的《RAID 入门指南》。

注：当您使用 Server Assistant 安装支持的操作系统时，Server Assistant 将相关的系统管理软件安装文件自动复制到硬盘驱动器上，并在桌面上放置“Install Server Administrator”（安装 Server Administrator）和“Delete Server Administrator Installation Files”（删除 Server Administrator 安装文件）图标。
注：您可以使用“Install Server Administrator”（安装 Server Administrator）图标来安装 Server Administrator，而不使用安装 CD。 在运行 Windows 的系统上，单击此图标显示标准安装界面。 在运行 Red Hat Enterprise Linux 或 SUSE LINUX Enterprise Server 的系统上，单击此图标会运行 Server Administrator 自定义安装脚本。 如果您不想安装 Server Administrator，可以通过单击“Delete Server Administrator Installation Files”（删除 Server Administrator 安装文件）图标来删除安装文件。 您确定要继续后，所有的 Server Administrator 文件（包括图标）都会被删除。
注：当安装 Windows 操作系统时，一些装有 4 GB 以上物理内存的硬件配置在安装操作系统后需要执行附加的步骤，以充分利用所有已安装的内存。 有关物理地址扩展 (PAE) 的详情，请参阅：
  • www.microsoft.com/windows2000/en/advanced/help/PAE_checklist.htm

  • www.microsoft.com/resources/documentation/windowsserv/2003/enterprise/proddocs/en-us/paex86_2.asp

    安装程序提供了“Express Setup”（快速设置）选项和“Custom Setup”（自定义设置）选项。 “Express Setup”（快速设置）选项（推荐）将自动安装管理系统所需的所有检测到的软件组件。 “Custom Setup”（自定义设置）选项使您能选择要安装的软件组件。 本说明文件中的步骤依据为“Express Setup”（快速设置）选项。 有关“Custom Setup”（自定义设置）的信息，请参阅《Dell OpenManage 安装和安全性用户指南》。

    注： 请参阅《安装和安全性用户指南》以获得自定义安装和其它内容详情。 Management Station 和 Managed System Software 可在相同或不同目录下安装。 您可以选择安装目录。
    注：请参阅 Installation and Server Management CD 上的安装自述文件以获得目前所支持系统的列表。 安装自述文件 readme_ins.txt 位于 CD 的 srvadmin/docs/readme 目录下
    注：如果系统上已安装了 4.2 或更早版本 (>=3.0) 的 Systems Management Software，则不能直接升级至 5.0 版。 要保留这些设置，您必须先升级至 4.3 版，然后再升级至 5.0 版以获得最新的软件更新。 4.3 版可在 Dell 支持站点 support.dell.com 上得到。
    注：在安装 Managed System Software 之前，请使用操作系统介质在管理系统上安装 SNMP 代理。

    为支持的 Microsoft Windows 操作系统安装 Managed System Software

    1. 使用管理员权限登录至运行支持的 Windows 操作系统的系统，以在该系统中安装管理系统组件。

    2. 验证系统已装有支持的浏览器。

    3. 如果系统已经安装 RAID 控制器并且您打算安装存储管理功能，请确保用于每个 RAID 控制器的设备驱动程序也已经安装。

    您可以在 Service and Diagnostic Utilities CD 上找到设备驱动程序。 您可以在 Windows 下验证设备驱动程序，方法是：在“My Computer”（我的电脑）上单击鼠标右键，选择“Manage”（管理），然后单击“Device Manager”（设备管理器）。

    1. 将 Installation and Server Management CD 插入到 CD 驱动器中。

    安装程序应当自动启动。 如果没有自动启动，请单击“Start”（开始）按钮，再单击“Run”（运行），然后键入 x:\srvadmin\windows\setup.exe（其中 x 是 CD 驱动器号）。 安装程序会启动前提条件检查公用程序。

    1. 查看所执行的前提条件检查结果，并解决好列出的任何情况。

    2. 单击“Install, Modify, Repair, or Remove Server Administrator”（安装、修改、修复或删除 Server Administrator）按钮。

    3. 在“Welcome”（欢迎）屏幕中单击“Next”（下一步）。

    4. 接受许可证协议，然后单击“Next”（下一步）。

    5. 选择“Express”（快速）设置，然后单击“Next”（下一步）。

    6. 单击“Install”（安装）继续安装过程。

    安装程序将根据您的硬件配置自动安装所有 Managed System Software。

    1. 安装完成后，单击“Finish”（完成），并在使用该软件前重新启动系统（若提示）。

    为支持的 Red Hat Enterprise Linux 操作系统或 SUSE LINUX Enterprise Server 操作系统安装 Managed System Software

    1. 以 root 身份登陆至运行支持的 Red Hat Enterprise Linux 或 SUSE LINUX Enterprise Server 操作系统的系统，以在该系统中安装管理系统组件。

      注：只有 PowerEdge 1800、1850、2800 和 2850 系统支持 SUSE LINUX Enterprise Server 操作系统。

    2. 将 Installation and Server Management CD 插入到 CD 驱动器中。

    3. 如果有必要，请使用 mount /dev/cdrom /media/cdrom 命令或一个类似命令（对于 Red Hat Enterprise Linux 4 和 SUSE LINUX Enterprise Server 9）或 mount /dev/cdrom /mnt/cdrom 或一个类似命令（对于 Red Hat Enterprise Linux 3）安装 CD。

    4. 从以下目录执行 srvadmin-install.sh 脚本：

    /srvadmin/linux/supportscripts ，命令如下：

    sh srvadmin-install.sh --express

    或

    sh srvadmin-install.sh -x

    该脚本将为系统配置安装典型的软件套件。

    注：您可以通过将 2>&1 | tee Ca /var/log/srvadmin.log 添加到以上外壳脚本命令中来记录 RPM 安装的输出。 由此得到的命令为： sh srvadmin-install.sh 2>&1|tee Ca /var/log/srvadmin.log
    1. 使用 sh srvadmin-services.sh start 命令启动服务。

    为支持的 Red Hat Enterprise Linux 操作系统或 SUSE LINUX Enterprise Server 操作系统安装 Management Station Software

    Management Station 软件套件的 BMC 管理公用程序和 RAC 组件可在运行 Red Hat Enterprise Linux 或 SUSE LINUX Enterprise Server 操作系统的管理站上使用。

    注：只有 PowerEdge 1800、1850、2800 和 2850 系统支持 SUSE LINUX Enterprise Server 操作系统。

    要在管理站上安装 BMC 管理公用程序，请执行下列步骤来启动安装程序：

    1. 以 root 身份登录至您想在其中安装 Management Station 组件的系统。

    2. 如果有必要，请使用 mount /dev/cdrom /media/cdrom 命令或一个类似命令（对于 Red Hat Enterprise Linux 4 和 SUSE LINUX Enterprise Server 9）或 mount /dev/cdrom /mnt/cdrom 或一个类似命令（对于 Red Hat Enterprise Linux 3）安装 CD。

    3. 浏览至 /linux/bmc 目录，并使用操作系统特定的 rpm 命令安装 BMC 软件：

      • 对于运行 Red Hat Enterprise Linux 的系统，使用 osabmcutil*-RHEL-*.rpm，例如：rpm -Uvh osabmcutil*-RHEL-*.rpm。

      • 对于运行 SUSE LINUX Enterprise Server 的系统，使用 osabmcutil*-SUSE-*.rpm，例如：rpm -Uvh osabmcutil*-SUSE-*.rpm。

    要安装 RAC Management Station 组件，请执行下列步骤以启动安装程序：

    1. 以 root 身份登录至您想在其中安装 Management Station 组件的系统。

    2. 如果有必要，请使用 mount /dev/cdrom /media/cdrom 命令或一个类似命令（对于 Red Hat Enterprise Linux 4 和 SUSE LINUX Enterprise Server 9）或 mount /dev/cdrom /mnt/cdrom 或一个类似命令（对于 Red Hat Enterprise Linux 3）安装 CD。

    3. 浏览至 /linux/rac 目录，并执行
      rpm -ivh *.rpm 命令。

    更新驱动程序和公用程序

    Service and Diagnostic Utilities CD 包含有可以用来更新系统的设备驱动程序和公用程序。 您必须将这些驱动程序和公用程序复制到软盘或硬盘才能使用它们。

    要获得最新的产品更新，如驱动程序和 Dell OpenManage 应用程序，请访问 Dell 支持网站 support.dell.com。

    注：您只能在运行 Windows 的系统上运行 Service and Diagnostic Utilities CD。 要为运行 Red Hat Enterprise Linux 或 SUSE LINUX Enterprise Server 操作系统的系统抽取驱动程序，您必须在运行 Windows 的系统上使用 Service and Diagnostic Utilities CD，然后将选定的文件复制到最终目标系统，或与最终目标系统共享选定的文件。
    1. 将 Service and Diagnostic Utilities CD 插入到运行 Windows 的系统之 CD 驱动器中。

    安装程序应当自动启动。 如果没有自动启动，请单击“Start”（开始）按钮，再单击“Run”（运行），然后键入 x:\setup.exe（其中 x 是 CD 驱动器号）。

    1. 选择系统，并从“Select Drivers/Utilities Set”（选择驱动程序/公用程序集）中选择您想更新的文件集，然后单击“Continue”（继续）。

    2. 在“Drivers and Utilities Page”（驱动程序和公用程序页），通过单击设备名称链接，选择您想更新的公用程序或驱动程序文件。

    系统提示您选择保存文件的位置。 这些文件是以压缩格式发送的。 抽取固件文件时通常需要软盘。

    1. 请解压缩这个文件。

    有关驱动程序和 Dell OpenManage 应用程序版本的其它信息，请参阅《Dell OpenManage Server Administrator 兼容性指南》。

    Dell PowerEdge Updates CD 仅同 Subscription Kit 一同发售，并通过 Server Update Utility (SUU) 和 Dell Update Packages (DUP) 为系统提供较新版软件。 供 Microsoft Windows 和 Linux 使用的 SUU 以单独的 CD 提供。

    1. 在运行 Windows、Red Hat Enterprise Linux 或 SUSE LINUX Enterprise Server 操作系统的系统上，将 Updates CD 插入到 CD 驱动器中。
    2. Server Update Utility 应当自动启动。 如果没有自动启动，请单击“Start”（开始）按钮，再单击“Run”（运行），然后键入 cmd，并按Enter键。 当命令提示符出现时，将目录更改至 CD 驱动器的根目录。 键入 suu -g（用于图形模式）或 suu -h（以显示帮助选项）。

    注：在 Red Hat Enterprise Linux 或 SUSE LINUX Enterprise Server 操作系统上，-g（图形模式）选项仅可在 X Window 系统下使用。

    1. 如果 GUI 初始化失败，您需要安装最新版本的 OpenIPMI 驱动程序。 键入 suu -g -installIPMI 来安装最近版本的 OpenIPMI 驱动程序。
       
      注：Server Administrator 的 Server Administrator Instrumentation Service 对 OpenIPMI 设备驱动程序有最低版本要求，以便提供基于 IPMI 的信息和功能性。 要求的最低版本根据操作系统来确定。 有关详情请参阅《安装和安全性用户指南》。

    2. 要在 GUI 中查看存储库比较，请单击左侧树中显示的服务器主机名。 SUU 将比较服务器与存储库中的组件版本。 对于服务器上每一种与 SUU 存储库当前版本一致的组件，SUU 将显示一个绿色复选标记图标。 对于服务器上的任何旧组件，凡是可使用 SUU 更新的，均显示一个黄色警告图标。 如果找到可更新的组件，屏幕右上角将启用“Update”（更新）按钮。

    3. 单击“Update”（更新）以应用为服务器列出的全部更新。 您不能有选择地应用更新。 SUU 会显示出一个对话框，请您确认更新列出的组件。 单击“Yes”（是）继续。

    4. “Update Success”（更新成功）窗口出现，确认 SUU 和 DUP 更新成功，并且可能提示您重新引导系统完成更新。

    请参阅《Dell OpenManage Server Update Utility 用户指南》以获得其它信息。
