Maximización del valor del sistema Dell™ PowerEdge™

Use los CD proporcionados para instalar y poner en marcha rápidamente el sistema. Obtendrá resultados confiables y consistentes y asegurará una operación sin problemas. Los CD proporcionan:

  • Instalación más eficiente del sistema operativo: reduce el tiempo requerido para la instalación de los sistemas operativos Microsoft® Windows®, Novell® NetWare® y Red Hat® Enterprise Linux, gracias a que guía al usuario a través de un proceso proceso comprensible y gradual.

  • Instalación y configuración: proporciona las herramientas necesarias para instalar y configurar el software y los sistemas Dell PowerEdge. Las herramientas permiten la configuración y descubrimiento automáticos de los adaptadores de red y controladores RAID proporcionados por Dell.

  • Utilidades, archivos controladores y diagnósticos: permiten ver y crear discos de diagnóstico y controladores de software de Dell.

  • Systems Management: instala las aplicaciones de software Dell OpenManage que sirven para obtener la máxima confiabilidad y disponibilidad de los sistemas. Estas aplicaciones proporcionan las herramientas necesarias para supervisar, diagnosticar y actualizar los sistemas de manera remota.

  • Documentación: permite acceso rápido a los documentos sobre el sistema, el systems management software, el controlador RAID y documentos periféricos en los idiomas admitidos.

NOTA: Para ver la documentación acerca de sistemas que ejecutan Red Hat Enterprise Linux, debe ejecutar index.htm desde el directorio del idioma correspondiente.

Verificación del contenido del paquete del servicio de suscripción de Dell OpenManage

El paquete del servicio de suscripción incluye, entre otros, los siguientes componentes:

  • El CD Dell PowerEdge Installation and Server Management es un CD de inicio que contiene las herramientas necesarias para configurar el sistema e instalar el sistema operativo. Este CD contiene los últimos productos del systems management software, incluso los servicios de acceso remoto, Storage Management y los diagnósticos de Dell OpenManage Server Administrator.

  • El CD Dell Systems Management Consoles contiene todos los productos más recientes de la consola de Dell systems management, entre ellos, Dell OpenManage IT Assistant.

  • El CD Dell PowerEdge Service and Diagnostic Utilities provee las herramientas necesarias para configurar el sistema, así como lo más reciente en BIOS, firmware, diagnósticos y controladores para el sistema optimizados por Dell .

  • El CD Dell PowerEdge Updates se incluye sólo con el paquete del servicio de suscripción y proporciona versiones nuevas del software para el sistema a través de Dell OpenManage Server Update Utility (SUU) y Dell Update Packages (DUP).

  • El CD Dell PowerEdge Documentation le ayuda a permanecer actualizado con la documentación para sistemas, productos del Systems Management Software, periféricos y controladores RAID.

La mayoría de estos CD además contienen archivos léame, que proveen la información más reciente sobre los productos.

Compare el contenido de la caja de accesorios del sistema con la lista de empaque o la factura que se incluyen con el sistema. Si algún componente está dañado o no está incluido, llame a Dell dentro de los 30 días siguientes a la fecha de factura, para obtener su reemplazo de manera gratuita. Para obtener más información, consulte "Obtención de asistencia técnica."

Introducción al Systems Management Software

El Dell OpenManage Systems Management Software es un conjunto de programas de aplicación para sistemas PowerEdge. Este software permite administrar el sistema por medio de supervisión, diagnóstico, notificación y acceso remoto proactivos.

Cada sistema que desee administrar con productos del software de Dell OpenManage recibe el nombre de sistema administrado. Las aplicaciones del Managed system son Server Administrator, Dell OpenManage Array Manager y el software del remote access controller (RAC). Una estación de administración se puede utilizar para administrar de manera remota, uno o más sistemas administrados desde una ubicación central.

Instalación del sistema operativo

Realice los siguientes pasos para determinar si el sistema tiene sistema operativo instalado:

  1. Compruebe que el tablero, el mouse y el monitor estén conectados al sistema, y encienda el sistema.

  2. Lea y acepte el Contrato de licencia de software para continuar.

El sistema se reinicia. Si aparece un mensaje indicando que no existen unidades de disco de inicio o que no se encontró un sistema operativo, entonces el sistema no tiene sistema operativo. Tenga el CD del sistema operativo a la mano y lleve a cabo el procedimiento para instalar un sistema operativo.

Si el sistema cuenta con un sistema operativo preinstalado no es necesario que realice el procedimiento de instalación. Localice el documento titulado Instrucciones de instalación del sistema operativo, que se proporcionó con el sistema, y siga dicho procedimiento para completar la instalación.

Realice los siguientes pasos para instalar un sistema operativo en su sistema:

  1. Inserte el CD Installation and Server Management y reinicie el sistema.

  2. Seleccione Configuración del servidor en la página principal de Server Assistant.

  3. Siga las instrucciones paso a paso para configurar el hardware e instalar el sistema operativo.

Para obtener información adicional acerca de cómo instalar RAID, consulte Para iniciar con RAID en el CD de documentación.

NOTA: Cuando se utiliza Server Assistant para instalar los sistemas operativos Windows o Red Hat Enterprise Linux, Server Assistant automáticamente copia los archivos relevantes de instalación de Server Administrator en el disco duro y coloca los iconos Instalar Server Administrator y Eliminar archivos de instalación de Server Administrator en el escritorio.
NOTA: Puede usar el icono Instalar Server Administrator para instalar Server Administrator sin el CD de instalación. En los sistemas que ejecutan Windows, si se hace clic en este icono se ejecutará la interfaz de instalación estándar. En los sistemas que ejecutan Red Hat Enterprise Linux, al hacer clic en este icono se ejecuta la instalación personalizada de Red Hat Enterprise Linux. Si no desea instalar Server Administrator, puede eliminar los archivos de instalación haciendo clic en el icono Eliminar archivos de instalación de Server Administrator. Después de confirmar que desea continuar, todos los archivos de Server Administrator, incluyendo los iconos se eliminan.
NOTA: Al instalar el sistema operativo Windows, algunas configuraciones de hardware con más de 4 GB de memoria física instalada requieren de pasos adicionales después de la instalación del sistema operativo para que puedan utilizar completamente toda la memoria instalada. Para obtener más información acerca de la extensión de dirección física (PAE), consulte:

www.microsoft.com/windows2000/en/advanced/help/PAE_checklist.htm

www.microsoft.com/resources/documentation/windowsserv/2003/enterprise/proddocs/en-us/paex86_2.asp

Instalación del Managed System Software en sistemas operativos Microsoft Windows admitidos

  1. Inicie sesión con privilegios de administrador en el sistema que ejecuta el sistema operativo Windows admitido, donde desea instalar los componentes del Managed System.

  2. Compruebe que el sistema tenga instalado un explorador admitido.

  3. Si tiene un controlador RAID instalado en el sistema y planea instalar la función Storage Management, asegúrese que también estén instalados los controladores de dispositivos de cada controlador RAID.

En el CD Service and Diagnostic Utilities encontrará controladores de dispositivos. Puede comprobar el controlador de dispositivo en Windows haciendo clic con el botón derecho del mouse en Mi PC, seleccionando Administrar y luego haciendo clic en Administrador de dispositivos.

  1. Inserte el CD Installation and Server Management en la unidad de CD.

El programa de configuración debe iniciar automáticamente. En caso contrario, haga clic en el botón Inicio, haga clic en Ejecutar y luego escriba x:\srvadmin\windows\setup.exe (donde x es la letra de la unidad de CD). El programa de configuración inicia la utilidad de verificador de prerrequisitos.

  1. Revise los resultados de las verificaciones de prerrequisitos realizadas y corrija las condiciones que aparezcan en la lista.

  2. Haga clic en el botón Instalar, Modificar, Reparar o Eliminar Server Administrator.

  3. Haga clic en Siguiente en la ventana de Bienvenida.

  4. Acepte el Contrato de licencia y haga clic en Siguiente.

  5. Seleccione Configuración rápida y haga clic en Siguiente.

  6. Haga clic en Instalar para continuar con el proceso de instalación.

El programa de instalación automáticamente instala todo el Managed System Software para la configuración de hardware.

  1. Cuando termine la instalación, haga clic en Terminar y, si el programa se lo pide, reinicie el sistema antes de usar el software.

Instalación del software Managed System para los sistemas operativos Red Hat Enterprise Linux admitidos

  1. Inicie sesión como "root" en el sistema que ejecuta un sistema operativo Red Hat Enterprise Linux admitido, donde desea instalar los componentes de Managed System.

  2. Inserte el CD Installation and Server Management en la unidad de CD.

  3. En caso necesario, monte el CD utilizando el comando mount /mnt/cdrom u otro comando similar.

  4. Si planea instalar el software de DRAC III, DRAC III/XT, ERA o ERA/O, vaya al directorio /srvadmin/linux/express-install-with-RAC3. Si planea instalar el software de DRAC 4, vaya al directorio /srvadmin/linux/express-install-with-RAC4. Luego, desde cualquier directorio, ejecute el comando RPM siguiente:
    rpm –ihv *.rpm

  5. Si no está seguro de cuál es el RAC que está instalado en el sistema, entonces ejecute la secuencia de comandos srvadmin-install.sh desde el directorio /srvadmin/linux/supportscripts como sigue:
    sh srvadmin-install.sh --express
    or
    sh srvadmin-install.sh -x

La secuencia de comandos instalará el software RAC. Si no hay un dispositivo, la secuencia de comandos no va a instalar el software RAC.

  1. Inicie los servicios con el comando de inicio sh srvadmin-services.sh.

NOTA: Puede registrar la salida de la instalación RPM agregando 2>&1 | tee –a /var/log/srvadmin.log al comando RPM mencionado anteriormente. El comando que resulta sería:
rpm –ihv *.rpm 2>&1|tee –a /var/log/srvadmin.log

Instalación del Managed System Software en sistemas operativos Novell NetWare admitidos

  1. Inserte el CD Installation and Server Management en la unidad de CD del sistema que ejecuta NetWare 6.5.

El CD se carga automáticamente.

  1. Si no aparece el nombre de volumen, escriba volumes cuando la consola se lo pida y presione <Entrar>.

Aparecerán todos los volúmenes disponibles.

  1. Registre el nombre de volumen para la unidad de CD.

  2. Escriba load nwconfig y presione <Entrar>.

  3. En la pantalla Configuración de NetWare, bajo Opciones de configuración, seleccione Opciones del producto y presione <Entrar>.

  4. En el menú Otras acciones de instalación, seleccione Instalar un producto no listado y presione <Entrar>.

NOTA: Después de presionar <Entrar>, es posible que aparezca un cuadro de diálogo que liste las rutas de acceso seleccionadas anteriormente. Presione <Esc> para cerrar este cuadro de diálogo.

Aparece un mensaje predeterminado, informándole que el producto se va a instalar desde la unidad de disco A:.

  1. Presione <F3> para especificar una ruta de acceso diferente para la instalación .

  2. Borre A: y escriba el nombre del volumen y una ruta de acceso:

nombre_del_volumen:srvadmin\netware

donde nombre_del_volumen es el nombre del volumen asignado a la unidad de CD.

NOTA: No puede introducir espacios en la ruta de acceso.
  1. Después de escribir la ruta de acceso, presione <Entrar> para continuar.

Aparece el mensaje siguiente :

Indicate which file groups you want installed.

(Indique los grupos de archivos que desea instalar.)

Server Administrator no está listado como opción porque se instala de manera predeterminada.

  1. Seleccione los componentes que desea instalar.

NOTA: Los elementos seleccionados para su instalación tienen una X a un lado. Al presionar <Entrar> al lado de un elemento, se selecciona ese elemento. Al presionar la barra espaciadora al lado de un elemento, cambia la selección entre seleccionado y no seleccionado.
  1. Presione <F10> para aceptar los componentes que seleccionó.

  2. La pantalla de resumen aparece y éste es el último punto en que el usuario puede detener la instalación. Revise la información del resumen y presione <Entrar> para continuar o <Esc> para anular la instalación.

Inicia la instalación de Server Administrator y los componentes que seleccionó. Aparecen mensajes de estado.

Aparece la utilidad de seguridad de Systems Management preguntándole si desea cambiar la configuración de seguridad predeterminada para las operaciones Set de SNMP.

NOTA: La contraseña root predeterminada es calvin.
  1. Seleccione una de las siguientes opciones:

    • Presione <y> para cambiar la contraseña root predeterminada.

Escriba la contraseña nueva y luego vuelva a escribirla para confirmar.

    • Presione <n> o <q> para salir de la utilidad de seguridad y restablecer la contraseña de root a su valor predeterminado.

Continúa la instalación de Server Administrator y los componentes que seleccionó. Aparecen mensajes de estado. Cuando termina la instalación aparece un cuadro de diálogo informándole que la instalación está completa.

  1. Presione <Entrar>.

  2. Presione <Esc> dos veces y <Entrar> una vez para regresar al símbolo de la consola y salir del programa de configuración de NetWare.

Instalación del Systems Management Software en una estación de administración

La instalación rápida instala IT Assistant, la consola RAC y la Utilidad de administración de BMC. La instalación rápida instala todos estos componentes. Si sólo desea instalar uno o algunos de ellos, entonces consulte las secciones de instalación personalizada de la Guía del usuario de instalación y seguridad de Dell OpenManage.

NOTA: Para Windows, las management station applications se ejecutan sólo en sistemas ejecutando Windows XP Professional, Windows 2000 Professional (SP4), la familia Windows Server 2003 (incluyendo Small Business Server, SBS) y la familia Windows 2000 Server (incluyendo SBS), además pueden administrar sistemas ejecutando los sistemas operativos admitidos Windows, NetWare y Red Hat Enterprise Linux. IT Assistant no es admitido en los sistemas operativos Windows Server 2003 SBS o Windows 2000 Server SBS.
NOTA: Sólo la consola RAC y la utilidad de administración de BMC son admitidas para los sistemas operativos Red Hat Enterprise Linux.

Instalación del software Management Station en el sistema operativo Red Hat Enterprise Linux

La utilidad de administración de BMC y los componentes de RAC del paquete de software Management Station se pueden usar en una estación de administración que ejecute Red Hat Enterprise Linux.

Para instalar la utilidad de administración de BMC en la estación de administración, realice los pasos a continuación para iniciar el programa de instalación:

  1. Inicie sesión como root en el sistema en donde desea instalar los componentes de management station.

  2. En caso necesario, monte el CD utilizando el comando mount /mnt/cdrom u otro comando similar.

  3. Vaya al directorio /linux/bmc y ejecute el comando rpm -ivh *.rpm.

Para instalar el componente Management Station de RAC, realice los pasos siguientes para ejecutar el programa de instalación:

  1. Inicie sesión como root en el sistema en donde desea instalar los componentes de management station.

  2. En caso necesario, monte el CD con el comando mount /mnt/cdrom u otro comando similar.

  3. Vaya al directorio /linux/rac y ejecute el comando rpm -ivh *.rpm.

Actualización de archivos controladores y utilidades

El CD Service and Diagnostic Utilities contiene controladores de dispositivos y utilidades que puede utilizar para actualizar el sistema. Para poder utilizarlos, debe copiar los controladores y las utilidades ya sea a un disco o a la unidad de disco duro.

Para obtener las actualizaciones más recientes de los productos como BIOS, firmware, archivos controladores y aplicaciones de Dell OpenManage, visite el sitio web de asistencia Dell en support.dell.com.

NOTA: El CD Service and Diagnostic Utilities sólo se puede ejecutar en sistemas que ejecutan Windows. Para extraer archivos controladores para sistemas que ejecutan Red Hat Enterprise Linux o Netware, debe utilizar el CD Service and Diagnostic Utilities en un sistema que ejecute Windows, y luego copiar o compartir los archivos seleccionados con el sistema de destino final.
  1. Inserte el CD Service and Diagnostic Utilities en la unidad de CD de un sistema que ejecute Windows.

El programa de configuración debe iniciar automáticamente. En caso contrario, haga clic en el botón Inicio, haga clic en Ejecutar y luego escriba x:\setup.exe (donde x es la letra de la unidad de CD).

  1. Seleccione el sistema, y en Seleccionar controladores/utilidades, seleccione el conjunto de archivos que desea actualizar y luego haga clic en Continuar.

  2. Desde la Página de utilidades y controladores, elija el archivo que tenga la utilidad o el controlador que desee actualizar, haciendo clic en el vínculo del nombre del dispositivo.

El sistema le preguntará en dónde desea guardar el archivo. Los archivos se guardan en formato comprimido. Los archivos de BIOS y firmware normalmente requieren discos flexibles para la extracción.

  1. Descomprima el archivo.

Consulte la Guía de compatibilidad de Dell OpenManage Server Administrator para obtener información adicional sobre las versiones de las aplicaciones de BIOS, firmware, controladores y Dell OpenManage.

El CD Dell PowerEdge Updates se envía sólo con el paquete de suscripción y proporciona versiones nuevas del software para el sistema a través de Server Update Utility (SUU) y Dell Update Packages (DUP).

  1. Inserte el CD Updates en la unidad de CD en un sistema que ejecute el sistema operativo Windows o Red Hat.

  2. La Server Update Utility debe iniciar automáticamente. De lo contrario, haga click en el botón de Inicio, y haga clic en Ejecutar, escriba cmd, y presione Entrar. Cuando aparezca el símbolo del sistema, cambie de directorio al directorio raíz de la unidad de CD. Escriba suu -g (para el modo gráfico) o suu -h (para mostrar las opciones de ayuda).

NOTA: Para sistemas Red Hat Enterprise Linux, la opción -g (modo gráfico) sólo se puede utilizar desde el sistema X Window.
  1. Para ver una comparación de repositorios en la GUI, haga clic en el nombre del host del servidor mostrado en el árbol a la izquierda. La SUU compara lo que está en el servidor con las versiones de los componentes disponibles en su repositorio. Para cada componente del servidor que esté al corriente con la versión del repositorio de SUU, éste último mostrará un icono de marca de verificación verde. Para cualquier componente en el servidor que no esté al corriente y pueda ser actualizado utilizando SUU, se mostrará un icono de advertencia amarillo. Si se encuentran componentes actualizables, se activará un botón de Actualizar en la esquina superior derecha de la pantalla.

  2. Haga clic en Actualizar para aplicar las actualizaciones que estén listadas para el servidor. No se pueden aplicar actualizaciones de manera selectiva. La SUU mostrará un cuadro de diálogo pidiéndole que confirme que desea actualizar los componentes listados. haga clic en Sí para continuar.

  3. Aparece la ventana de Actualización satisfactoria, confirmando el éxito de la actualización mediante SUU y DUP, y puede solicitarle reiniciar el sistema para completar la actualización.

Dell Update Packages:

Cuando haga clic en Actualizar en la SUU, ésta determinará cuál de los Dell Update Packages son aplicables para el sistema y los instalará automáticamente.

Consulte la Guía del usuario de Dell OpenManage Server Update Utility para más información.
