Présentation générale

Vous pouvez installer le logiciel Managed System Software de plusieurs façons. Le CD Dell PowerEdgeInstallation and Server Management offre des scripts IPS d'installation pour installer, mettre à niveau et désinstaller Dell OpenManage™ Server Administrator et d'autres composants du logiciel Managed System Software sur votre système géré. Vous pouvez également installer Managed System Software sur plusieurs systèmes en utilisant une installation automatique sur le réseau.

REMARQUE : Consultez le fichier lisez-moi readme_ins.txt du CD Dell PowerEdge Installation and Server Management pour obtenir la liste des systèmes actuellement pris en charge.Si une version précédente du logiciel Systems Management Software est détectée, vous devez d'abord la désinstaller. Une fois la désinstallation terminée, vous devez redémarrer le système pour pouvoir installer la dernière version du logiciel de gestion de systèmes.

Installation automatique et en arrière plan indiquée par le script

Vous pouvez utiliser le CD Dell PowerEdge Installation and Server Management pour effectuer une installation en arrière plan et automatique de Server Administrator suivant le script sur un système qui fonctionne sous un système d'exploitation Novell® Netware® pris en charge.

Avant de commencer

  • Consultez les spécifications pour l'installation pour vous assurer que votre système correspond au moins aux spécifications minimales.

  • Lisez le Guide de compatibilité de Server Administrator. Ce document fournit des informations concernant la compatibilité de l'installation et du fonctionnement de Server Administrator pour différentes plates-formes matérielles exécutant un système d'exploitation Microsoft Windows, NetWare® ou Red Hat® Enterprise Linux pris en charge.

  • Lisez le fichier readme_ins.txt du programme d'installation Dell OpenManage sur le CD Dell PowerEdge Installation and Server Management Ce fichier contient les informations les plus récentes sur les nouvelles fonctionnalités, les corrections et les spécifications matérielles, ainsi que des informations sur les problèmes reconnus.

  • Lisez le fichier lisez-moi de Server Administrator sur le CD Dell PowerEdge Installation and Server Management. Ce fichier contient les informations les plus récentes sur les versions de logiciel, de micrologiciel et de pilotes, ainsi que des informations sur les problèmes reconnus.

  • Lisez les instructions d'installation de votre système d'exploitation.

Spécifications pour l'installation

Les sections suivantes décrivent les spécifications générales pour l'installation de Managed System Software.

Versions des systèmes d'exploitation prises en charge

Managed System Software prend en charge, au minimum, les systèmes d'exploitation suivants :

  • Novell NetWare version 6.5

REMARQUE : Consultez le fichier lisez-moi de Server Administrator sur le CD Dell PowerEdge Installation and Server Management ou le Guide de compatibilité de Dell OpenManage Server Administrator sur le CD de documentation pour obtenir la liste détaillée la plus récente des services de Server Administrator qui sont offerts pour chaque système d'exploitation pris en charge.

Spécifications système

Managed Systems Software doit être installé sur chaque système à gérer. Vous pouvez ensuite gérer localement ou à distance, en utilisant un navigateur Web pris en charge, chaque système exécutant Managed System Software.

Spécifications du système géré

  • Une des versions prises en charge du système d'exploitation.

  • Un minimum de 64 Mo de RAM.

  • Un minimum de 256 Mo d'espace libre sur le disque dur.

  • Les droits d'administrateur.

  • Une connexion TCP/IP sur le système géré et sur le système distant pour faciliter la gestion du système distant.

  • La norme de protocole simplifié de gestion de réseau (SNMP).

  • Une souris, un clavier et un moniteur pour gérer un système localement. Le moniteur doit avoir une résolution d'écran d'au moins 800 x 600. Le paramètre de résolution d'écran conseillé est 1024 x 768.

  • Pour utiliser le service d'accès à distance de Server Administrator, un Remote Access Controller (RAC) doit être installé sur le système à gérer. Consultez le Guide d'utilisation de Dell Remote Access Controller 4 ou le Guide d'utilisation de Dell Embedded Remote Access Controller/MC pour connaître toutes les spécifications logicielles et matérielles.

Normes de protocole de gestion de systèmes prises en charge

Une norme de protocole de gestion de systèmes prise en charge doit être installée sur le système géré avant d'installer Managed System Software. Managed System Software prend en charge uniquement la norme de gestion de système SNMP sur les systèmes d'exploitation NetWare pris en charge. SNMP est toujours installé. Les normes CIM et WMI ne sont pas disponibles.

REMARQUE : Pour des informations sur l'installation d'une norme de protocole de gestion de systèmes prise en charge sur votre système géré, consultez la documentation de votre système d'exploitation.

Procédures d'installation

Cette section explique comment installer, mettre à niveau et désinstaller Managed System Software sur un système qui exécute un système d'exploitation Netware pris en charge.

Installation et mise à niveau de Managed System Software

Cette section explique comment installer et mettre à niveau Managed System Software en utilisant les options d'installation suivantes :

  • Utilisez le programme d'installation sur le CD Dell PowerEdge Installation and Server Management pour installer ou mettre à jour Server Administrator et les autres logiciels Managed System Software.

  • Utilisez le processus d'installation en arrière plan indiqué par le script pour installer Server Administrator et les autres logiciels de Managed System sur plusieurs systèmes.

Prérequis pour l'installation ou la mise à niveau de Managed System Software

Les prérequis pour l'installation de Managed System Software sont les suivants :

  • Vous devez avoir des privilèges d'administrateur.

  • L'agent SNMP fourni avec le système d'exploitation doit être installé sur le système géré. Si vous voulez utiliser des agents de prise en charge pour l'agent SNMP, vous devez installer la prise en charge par le système d'exploitation de la norme SNMP avant d'installer Server Administrator. Pour des informations supplémentaires sur l'installation de SNMP, consultez les instructions d'installation du système d'exploitation que vous exécutez sur votre système.

REMARQUE : L'installation de Managed System Software n'est pas prise en charge sur les systèmes contenant une version du logiciel Systems Management Software antérieure à la version 4.3. Si une version précédente du logiciel Systems Management Software est détectée, vous devez d'abord désinstaller Systems Management Software sur le système.

Installation et mise à niveau de Managed System Software avec le CD Dell PowerEdge Installation and Server Management

  1. Insérez le CD Dell PowerEdge Installation and Server Management dans le lecteur de CD de votre système.

Le CD se charge automatiquement et tous les noms de volume s'affichent.

Si le nom du volume ne s'affiche pas, tapez volume à l'invite de la console et appuyez sur <Entrée>.

  1. Enregistrez le nom de volume du lecteur de CD.

  2. Tapez load nwconfig et appuyez sur <Entrée>.

  3. Dans l'écran Configuration de NetWare, sous la rubrique Options de configuration, sélectionnez Options de produit et appuyez sur <Entrée>.

  4. Sur le menu Autres actions d'installation, sélectionnez Installer un produit non répertorié et appuyez sur <Entrée>.

REMARQUE : Une boîte de dialogue répertoriant les chemins sélectionnés plus tôt apparaîtra peut-être quand vous appuyez sur <Entrée>. Appuyez sur <Échap> pour la fermer.

Un message par défaut indique ensuite que le produit doit être installé à partir du lecteur A.

  1. Appuyez sur <F3> pour définir un chemin d'installation différent.
    Effacez A: et tapez nom_du_volume:srvadmin\netware nom_du_volume étant le nom du volume assigné au lecteur de CD.

REMARQUE : Vous ne pouvez pas entrer d'espace dans le chemin d'accès.
  1. Après avoir entré le chemin, appuyez sur <Entrée> pour continuer.
    Un message est affiché :
    Indicate which file groups you want installed.

    (Indiquez quels groupes de fichiers vous voulez installer.)

  2. Sélectionnez tous les autres composants que vous voulez installer. Server Administrator n'est pas compris dans la liste de groupes de fichiers car il est installé par défaut.

REMARQUE : Un X se trouve à côté des éléments sélectionnés pour être installés. Si vous appuyez sur <Entrée> à côté d'un élément, cet élément sera sélectionné. Si vous appuyez sur la barre d'espacement à côté d'un élément, vous activez et désactivez la sélection.
  1. Appuyez sur <F10> pour accepter les composants que vous avez sélectionnés.

  2. L'écran qui suit est l'écran de résumé. Examinez les informations de résumé et appuyez sur <Entrée> pour continuer ou sur <Échap> pour annuler l'installation.
    L'installation de Server Administrator et des composants sélectionnés commence.

  3. L'écran de l'Utilitaire de sécurité de la gestion des systèmes apparaît et vous demande si vous souhaitez changer les paramètres de sécurité par défaut pour les opérations Set SNMP.

  4. Sélectionnez l'une des options suivantes :

    • Appuyez sur <y> pour modifier le mot de passe par défaut de root.
      Tapez le nouveau mot de passe, puis retapez-le pour le confirmer.

REMARQUE : Le mot de passe par défaut pour root est calvin.
    • Appuyez sur <n> pour quitter l'utilitaire de sécurité et redéfinir le mot de passe de root sur sa valeur par défaut.

L'utilitaire de sécurité de Systems Management s'ouvre.

  1. Lorsque vous y êtes invité, appuyez sur <y> pour permettre les opérations set SNMP (écriture) ou <n> pour rejeter les opérations set SNMP (lecture seule), ou sur n'importe quelle touche pour continuer.

La procédure d'installation continue et des messages d'état apparaissent. Lorsque l'installation est terminée, une boîte de dialogue apparaît indiquant que l'installation est terminée.

  1. Appuyez sur <Entrée>.

  2. Appuyez deux fois sur <Échap> et une fois sur <Entrée> pour retourner à l'invite de la console et quitter le programme NWConfig.

Réalisation d'une installation ou d'une mise à niveau en arrière plan indiquée par le script de Managed System Software

Utilisez la procédure suivante pour installer Managed System Software en arrière plan pour la première fois ou pour remplacer la version de Managed System Software par une version plus récente.

  1. Insérez le CD Dell PowerEdge Installation and Server Management dans le lecteur de CD de votre système.

Le CD se charge automatiquement et le nom de volume s'affiche.

Si le nom du volume ne s'affiche pas, tapez volume à l'invite de la console et appuyez sur <Entrée> pour afficher tous les noms de volume.

  1. Enregistrez le nom de volume du lecteur de CD.

  2. Tapez la commande suivante et appuyez sur <Entrée> :

load nom_de_volume:\srvadmin\netware\ominss32.nlm [-noinstallweb] [-noam] [-norac]

Le script d'installation installe tous les fichiers nécessaires sur le système géré.

Vous pouvez afficher le contrat de licence (license.txt) dans le répertoire root du CD Dell PowerEdge Installation and Server Management.

Le paramètre –noinstallweb est optionnel et résultera dans une installation qui a seulement les composants de ligne de commande et ne sera pas accessible par un navigateur.

Le paramètre –noam est optionnel et résultera dans une installation qui n'a pas Dell OpenManage Array Manager.

Le paramètre –norac est optionnel et résultera dans une installation qui n'a pas de prise en charge logicielle de Dell Remote Access Controller.

Désinstallation de Managed System Software

Vous pouvez désinstaller Managed System Software sur votre système à partir du menu Options de produits dans l'utilitaire NWConfig ou en utilisant le script de désinstallation automatique fourni sur le CD Dell PowerEdge Installation and Server Management.

Désinstallation avec l'utilitaire NWConfig

  1. Tapez load nwconfig sur la console de la ligne de commande NetWare et appuyez sur <Entrée>.

  2. Sélectionnez Options de produits dans le menu Options de configuration.

  3. Sélectionnez Afficher/Configurer/Supprimer dans le menu Autres actions d'installation.

  4. Sélectionnez SRVADMIN dans le menu Produits actuellement installés.

  5. Appuyez sur <Supprimer> pour désinstaller Server Administrator.

  6. Appuyez deux fois sur <Échap> et une fois sur <Entrée> pour retourner à l'invite de la console et quitter le programme NWConfig.

Réalisation d'une désinstallation en arrière plan

Tapez omunins.ncf dans la ligne de commande de la console NetWare et appuyez sur <Entrée>. Le script de désinstallation désinstalle tous les fichiers de Managed System Software du système géré.

Remarque au moment du démarrage

Instrumentation Service au démarrage

Sur les systèmes gérés dotés du système d'exploitation NetWare, Instrumentation Service charge différents modules chargeables NetWare (NLM) au démarrage du système. Certains NLM peuvent être automatiquement désinstallés parce qu'ils ne sont pas nécessaires sur un système particulier. Le processus de désinstallation des NLM se produit environ deux minutes après le démarrage du système. Pendant la désinstallation, un message de déchargement de module s'affiche sur la console du système NetWare.
