Optimisation de votre système Dell™ PowerEdge™

Utilisez les CD fournis pour pouvoir utiliser votre système dans les plus brefs délais. Vous obtiendrez des résultats consistants et fiables et serez assuré d'un fonctionnement sans problèmes. Les CD fournissent :

  • Des outils d'installation de système d'exploitation optimisés qui réduisent le temps nécessaire à l'installation des systèmes d'exploitation Microsoft® Windows®, Red Hat®Enterprise Linux et SUSE® LINUX Enterprise Server en vous guidant tout au long d'étapes faciles à suivre.

Ces applications fournissent les outils nécessaires à la surveillance, le diagnostic et la mise à jour à distance de vos systèmes.

  • De la documentation traduite dans certaines langues sur les systèmes, le logiciel Systems Management Software, le contrôleur RAID et les périphériques.

Présentation générale de Systems Management Software

Le logiciel Dell OpenManage Systems Management est une suite de logiciels d'application pour les systèmes PowerEdge et Dell™ PowerVault™. Ces logiciels vous permettent de gérer votre système par surveillance active, diagnostic, notification et accès à distance.

Les systèmes que vous avez l'intention de gérer avec les logiciels Dell OpenManage sont appelés systèmes gérés. Les applications Managed System comprennent Server Administrator et le logiciel Remote Access Controller (RAC).

Une station de gestion peut servir à gérer un ou plusieurs systèmes gérés à distance à partir d'un emplacement central.

REMARQUE : La console Dell OpenManage Array Manager (pour Management Station) est disponible sous Windows uniquement si le logiciel Dell OpenManage Management Station précédent (avec la console Array Manager installée) est détecté. Il peut seulement être mis à niveau.

Installation de votre système d'exploitation à l'aide de Server Assistant

Effectuez les étapes suivantes pour déterminer si un système d'exploitation est installé sur votre système :

  1. Vérifiez que le clavier, la souris et le moniteur sont connectés à votre système et allumez ce dernier.

  2. Lisez et acceptez le contrat de licence logicielle pour continuer.

Si un message apparaît, indiquant qu'aucun disque d'amorçage n'existe ou qu'aucun système d'exploitation n'a été trouvé, c'est qu'il n'y a pas de système d'exploitation installé sur votre système. Prenez le CD de votre système d'exploitation et passez aux étapes suivantes.

Si un système d'exploitation a été pré-installé sur votre système, il n'est pas nécessaire de poursuivre cette procédure. Déterminez l'emplacement du document du système d'exploitation intitulé Instructions d'installation fourni avec votre système et suivez ces instructions pour effectuer l'installation.

Effectuez les étapes suivantes pour installer un système d'exploitation sur votre système :

  1. Insérez le CD Dell PowerEdge Installation and Server Management et redémarrez votre système.

Les options suivantes sont disponibles pendant le démarrage de système :

    • Server Assistant : démarre à partir du CD.

    • Server Assistant avec contrôle de support CD : contrôle si le CD est créé correctement ainsi que la validité du contenu.

    • Ne pas démarrer à partir du CD et démarrer à partir du disque dur : démarre à partir du disque dur et vérifie si un système d'exploitation est installé sur votre système. Si un système d'exploitation n'est pas installé sur votre système, démarre à partir du CD.

REMARQUE : Si vous ne sélectionnez pas d'option dans les 10 secondes, le système démarre automatiquement à partir du CD.
  1. Sélectionnez Configuration du serveur sur la page principale de Server Assistant.

  2. Suivez les instructions étape par étape pour configurer votre matériel et installer votre système d'exploitation.

Pour des informations supplémentaires sur l'installation du RAID, consultez la section Initiation au RAID du CD de documentation.

REMARQUE : Quand vous utilisez Server Assistant pour installer un système d'exploitation pris en charge, Server Assistant copie automatiquement les fichiers d'installation de Systems Management Software pertinents sur le disque dur et place les icônes Installer Server Administrator et Supprimer les fichiers d'installation de Server Administrator sur le bureau.
REMARQUE : Vous pouvez utiliser l'icône Installer Server Administrator pour installer Server Administrator sans le CD d'installation. Sur les systèmes fonctionnant sous Windows, le fait de cliquer sur cette icône affiche l'interface d'installation standard. Sur les systèmes exécutant Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server, le fait de cliquer sur cette icône exécute le script d'installation personnalisée de Server Administrator. Si vous ne voulez pas installer Server Administrator, vous pouvez supprimer les fichiers d'installation en cliquant sur l'icône Supprimer les fichiers d'installation de Server Administrator. Après avoir confirmé que vous voulez continuer, tous les fichiers de Server Administrator seront supprimés, y compris les icônes.
REMARQUE : Lors de l'installation du système d'exploitation Windows, quelques configurations matérielles avec plus de 4 Go de mémoire physique installée nécessitent des étapes supplémentaires après l'installation du système d'exploitation afin d'utiliser pleinement toute la mémoire installée.     Installation de Systems Management Software sur un système géré

    Le programme d'installation fournit une option d'installation rapide et une option d'installation personnalisée. L'option d'installation rapide (recommandée) installe automatiquement tous les composants logiciels détectés comme étant nécessaires à la gestion de votre système. L'option d'installation personnalisée vous permet de sélectionner quels composants logiciels vous voulez installer. La procédure reprise dans ce document est basée sur l'option d'installation rapide. Voir le Guide d'installation et de sécurité de Dell OpenManage pour des informations concernant l'installation personnalisée.

    AVIS : Dell OpenManage Array Manager n'est plus pris en charge. Si vous mettez à niveau un système (installé avec la version 4.3 ou ultérieure de Dell OpenManage) où Array Manager est installé, Array Manager sera supprimé pendant le processus de mise à niveau. Vous pouvez utiliser le service Server Administrator Storage Management à la place.
    REMARQUE : Voir le Guide d'installation et de sécurité pour l'installation personnalisée et d'autres détails. Management Station et Managed System Software peuvent être installés dans le même répertoire ou dans des répertoires différents. Vous pouvez sélectionner le répertoire de votre choix.
    REMARQUE : Consultez le fichier lisez-moi d'installation du CD Installation and Server Management pour obtenir la liste des systèmes pris en charge. Le fichier lisez-moi d'installation, readme_ins.txt, se trouve dans le répertoire srvadmin/docs/readme du CD
    REMARQUE : Si la version (>= 3.0) ou une version antérieure du logiciel Systems Management Software est installée sur votre système, vous ne pouvez pas directement mettre au niveau de la version 5.0. Pour préserver vos paramètres, vous devez d'abord mettre au niveau de la version 4.3, puis au niveau de la version 5.0 pour obtenir les dernières mises à jour de logiciels. La version 4.3 est disponible dans le site de support de Dell à support.dell.com.
    REMARQUE : Installez l'agent SNMP sur le système géré en utilisant votre système d'exploitation avant d'installer Managed System Software.
    REMARQUE : Pour éviter de démarrer à partir de Server Assistant, retirez le CD Installation and Server Management si vous redémarrez votre système.

    Installation de Managed System Software sur un système d'exploitation Microsoft Windows pris en charge

    1. Ouvrez une session avec des droits d'administrateur sur le système fonctionnant sous un système d'exploitation Windows pris en charge où vous voulez installer les composants Managed System .

    2. Vérifiez que votre système dispose d'un navigateur pris en charge.

    3. Si un contrôleur RAID est installé sur votre système et que vous avez l'intention d'installer la fonction Storage Management, assurez-vous que les pilotes de périphérique de chaque contrôleur RAID sont aussi installés.

    Des pilotes de périphérique sont disponibles sur le CD Service and Diagnostic Utilities. Vous pouvez vérifier le pilote de périphérique sous Windows en cliquant-droite sur Poste de travail, en sélectionnant Gérer et en cliquant ensuite sur Gestionnaire de périphériques.

    1. Insérez le CD Installation and Server Management dans le lecteur CD.

    Le programme de configuration devrait démarrer automatiquement. Si ce n'est pas le cas, cliquez sur le bouton Démarrer, puis sur Exécuter et tapez ensuite x:\srvadmin\windows\setup.exe (x étant la lettre représentant votre lecteur de CD). Le programme d'installation démarre l'utilitaire de vérification des prérequis.

    1. Vérifiez les résultats des vérifications des prérequis effectuées et résolvez tous les problèmes énumérés.

    2. Cliquez sur le bouton Installer, Modifier, Réparer ou Supprimer Server Administrator.

    3. Cliquez sur Suivant dans la fenêtre d'accueil.

    4. Acceptez le contrat de licence et cliquez sur Suivant.

    5. Sélectionnez Installation rapide et cliquez sur Suivant.

    6. Cliquez sur Installer pour poursuivre la procédure d'installation.

    Le programme de configuration installe automatiquement tous les composants Managed System Software qui s'appliquent à votre configuration matérielle.

    1. Une fois l'installation effectuée, cliquez sur Terminer et redémarrez votre système (si vous êtes invités à le faire) avant d'utiliser le logiciel.

    Installation de Managed System Software pour les systèmes d'exploitation Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server pris en charge

    1. Ouvrez une session en tant que « root » sur le système fonctionnant sous le système d'exploitation Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server où vous souhaitez installer les composants Managed System.

      REMARQUE : Le système d'exploitation SUSE LINUX Enterprise Server est uniquement pris en charge sur les systèmes PowerEdge 1800, 1850, 2800 et 2850.

    2. Insérez le CD Installation and Server Management dans le lecteur CD.

    3. Si nécessaire, montez le CD avec la commande mount/dev/cdrom/media/cdrom ou une commande semblable (pour Red Hat Enterprise Linux 4 et SUSE LINUX Enterprise Server 9) OU mount/dev/cdrom/mnt/cdrom ou une commande semblable (pour Red Hat Enterprise Linux 3)

    4. Exécutez le script srvadmin-install.sh à partir du

    répertoire /srvadmin/linux/supportscripts comme suit :

    sh srvadmin-install.sh --express

    ou

    sh srvadmin-install.sh -x

    Le script installera la suite logicielle typique pour votre configuration de système.

    REMARQUE : Vous pouvez journaliser les résultats de l'installation RPM en ajoutant 2>&1 | tee –a /var/log/srvadmin.log à l'exécution du script d'environnement ci-dessus. La commande qui en résulte sera sh srvadmin-install.sh 2>&1|tee -a /var/log/srvadmin.log
    1. Démarrez les services avec la commande sh srvadmin-services.sh start.

    Installation de Management Station Software pour les systèmes d'exploitation Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server pris en charge.

    Pour installer l'utilitaire de gestion du contrôleur BMC sur la station de gestion, effectuez les étapes suivantes pour lancer le programme d'installation :

    Mise à jour des pilotes et des utilitaires

    Le CD Service and Diagnostic Utilities contient des pilotes de périphériques et des utilitaires que vous pouvez utiliser pour mettre à jour votre système. Vous devez copier les pilotes et les utilitaires sur une disquette ou sur le disque dur pour pouvoir les utiliser.

    Pour les dernières mises à jour de produits, comme les pilotes et les applications Dell OpenManage, rendez-vous sur le site Web de support de Dell à l'adresse support.dell.com.

    REMARQUE : Vous ne pouvez exécuter le CD Service and Diagnostic Utilities que sur les systèmes qui fonctionnent sous Windows. Pour extraire des pilotes pour les systèmes qui fonctionnent sous Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server, vous devez utiliser le CD Service and Diagnostic Utilities sur un système fonctionnant sous Windows puis copier ou partager les fichiers sélectionnés sur le système de destination finale.
    1. Insérez le CD Service and Diagnostic Utilities dans le lecteur de CD d'un système fonctionnant sous Windows.

    Le programme de configuration devrait démarrer automatiquement. Si ce n'est pas le cas, cliquez sur le bouton Démarrer puis sur Exécuter et tapez x:\setup.exe (x étant la lettre de lecteur de votre lecteur de CD) .

    1. Sélectionnez le système et sélectionnez l'ensemble de fichiers à mettre à jour dans Sélectionnez un jeu de pilotes/utilitaires, puis cliquez sur Continuer.

    2. Sur la page Pilotes et utilitaires, sélectionnez l'utilitaire ou le pilote que vous voulez mettre à jour en cliquant sur le lien du nom de périphérique.

    Le système vous demande l'emplacement d'enregistrement du fichier. Les fichiers sont fournis en format compressé. Les fichiers du micrologiciel nécessitent normalement des disquettes pour pouvoir être extraits.

    1. Décompressez le fichier.

    Consultez le Guide de compatibilité de Dell OpenManage Server Administrator pour des informations supplémentaires sur les pilotes et les versions de l'application Dell OpenManage.

    Les CD Dell PowerEdge Updates sont uniquement expédiés avec le kit d'abonnement et fournit des versions plus récentes de logiciels pour votre système via Server Update Utility (SUU) et Dell Update Packages (DUP).SUU est livré sur des CD séparés pour Microsoft Windows et Linux.

    1. Insérez le CD Updates dans le lecteur de CD d'un système exécutant le système d'exploitation Windows, Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server .
    2. L'utilitaire Server Update Utility devrait démarrer automatiquement. S'il ne le fait pas, cliquez sur le bouton Démarrer, cliquez sur Exécuter, tapez cmd et appuyez sur Entrée. Quand la l'invite de commande apparaît, modifiez le répertoire pour vous positionner sur la racine du lecteur de CD. Tapez suu -g (pour le mode graphique) ou suu -h (pour afficher les options d'aide).

    REMARQUE : Sur les systèmes d'exploitation Red Hat Enterprise Linux ou SUSE LINUX Enterprise Server , l'option -g (mode graphique) ne peut être utilisée qu'à partir du système de X Window.

    1. Si l'initialisation de l'interface utilisateur échoue, vous devez installer la dernière version du pilote OpenIPMI. Tapez suu -g-installIPMI pour installer la dernière version d'OpenIPMIdriver.
       
      REMARQUE : Server Administrator Instrumentation Service exige une version minimale du pilote de périphérique OpenIPMI pour fournir des informations et des fonctionnalités IPMI. La version minimale exigée est définie en fonction du système d'exploitation. Voir le Guide d'installation et de sécurité pour plus d'informations.

    2. Pour afficher une comparaison de répertoires dans l'interface, cliquez sur le nom d'hôte de votre serveur affiché à gauche dans l'arborescence. SUU compare ce qui se trouve sur votre serveur avec les versions de composants disponibles dans son espace de stockage. Pour chaque composant de votre serveur à jour par rapport à la version de l'espace de stockage SUU , SUU indique une icône verte. Pour n'importe quel composant de votre serveur qui est dépassé et peut être mis à jour en utilisant SUU, une icône d'avertissement jaune s'affiche. Si des composants pouvant être mis à jour sont trouvés, un bouton Mettre à jour est activé dans le coin supérieur droit de l'écran.

    3. Cliquez sur Mettre à jour pour appliquer toutes les mises à jour qui sont énumérées pour votre serveur. Vous ne pouvez pas appliquer des mises à jour de manière sélective. SUU présente une boîte de dialogue demandant la confirmation de la mise à jour des composants énumérés. Cliquez sur Oui pour continuer.

    4. La fenêtre Mise à jour réussie apparaît, confirmant la réussite de la mise à jour pour SUU et DUP, et peut vous inviter à redémarrer votre système pour terminer la mise à jour.

    Dell Update Packages :

    Quand vous cliquez sur Mise à jour dans SUU, SUU déterminera quels progiciels Dell Update Packages s'appliquent à votre système et les installera automatiquement.

    Voir le Guide d'utilisation de Dell OpenManage Server Update Utility pour des informations supplémentaires.
